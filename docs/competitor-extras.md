# vbuff - What Else Competitors Have (feature-gap sweep)

> **122** additional / advanced / recent features found across competitor tools beyond the core catalog, grouped by area, with a suggested priority for vbuff.


## AI / ML clipboard features

1. **Semantic / meaning-based history search** `[v1]` - Search clipboard history by intent or vague description rather than exact keywords, using on-device embeddings that rerank clips by semantic similarity. _(seen in: Deck, Recopy, ClipMacs, CopyMagic)_
2. **OCR on copied images + searchable extracted text** `[v1]` - Automatically runs OCR on every copied screenshot/image so the text inside becomes part of searchable history. _(seen in: Clipory, Deck, Recopy, NanoClip, ToolPiper)_
3. **AI summarize / TL;DR of a clip** `[v1]` - Condense a long copied article or report into a one-line or short summary on demand or automatically. _(seen in: Recopy, ClipboardAI, Raycast AI, Paste (via Apple Intelligence/MCP))_
4. **AI rewrite / proofread / tone shift of a clip** `[v1]` - Rewrite copied text professionally, casually, or concisely, and proofread or refine it without leaving the clipboard UI. _(seen in: Recopy, Paste (Apple Intelligence), Raycast AI)_
5. **AI translate a clip** `[v1]` - Translate a copied snippet into another language inline from the clipboard manager. _(seen in: Recopy, Raycast AI Translate, ClipZ, Paste)_
6. **Ask-AI-about-the-clip (natural language action on copied item)** `[v1]` - Type a free-form instruction like "summarize what I just copied" or "translate this to French" that the AI executes against the current/selected clip. _(seen in: Raycast (@clipboard / Ask Clipboard, @selected-text), Recopy AI Actions)_
7. **On-copy smart categorization into typed buckets** `[MVP]` - Every copied item is auto-filed the instant it is copied into categories such as text, link, code, email, phone, address, image, color. _(seen in: ClipboardAI, ClipZ, ToolPiper, Paste)_
8. **Content recognition / data-type detection** `[MVP]` - Recognizes structured types within a clip (URLs, emails, IPs, tokens, code, colors) to drive contextual filters and per-type actions. _(seen in: Deck, ClipboardAI, Raycast)_
9. **OTP / 2FA code detection and surfacing** `[v2]` - Detects one-time passwords in copied text and flags/surfaces them for quick paste while marking them sensitive. _(seen in: ClipboardAI)_
10. **Sensitive-data detection and auto-protection** `[v1]` - Automatically detects passwords, tokens, SSNs and similar in clips and applies extra protection (hide/exclude/lock) instead of storing them plainly. _(seen in: ClipboardAI)_
11. **AI image description / captioning / alt-text for copied images** `[v2]` - Generates a natural-language description or alt-text caption for a copied image so it is searchable and screen-reader friendly. _(seen in: NanoClip, ClipMacs)_
12. **On-copy math / unit / percentage solving** `[v2]` - When you copy a math expression, unit conversion, or percentage, the AI computes the result automatically and offers it for paste. _(seen in: ClipboardAI)_
13. **List / checklist parsing from copied text** `[future]` - Detects a copied list and converts it into an interactive checklist with progress tracking. _(seen in: ClipboardAI)_
14. **Code explanation / key-info extraction action** `[v2]` - AI actions that explain a copied code snippet or pull out key facts (names, numbers, dates) from a copied blob. _(seen in: Recopy AI Actions, Raycast AI)_
15. **Clipboard as AI memory layer via MCP server** `[future]` - Exposes clipboard history through a local MCP server so external AI tools (Claude, Cursor, Codex) can search clips and build pinboards/drafts from them. _(seen in: Paste (Paste MCP))_
16. **Custom AI instructions / commands over clips** `[v2]` - User-defined persistent instructions or reusable AI commands (with placeholders like {{selectedText}}) that steer how copied content is transformed. _(seen in: Recopy (Custom AI Instructions), Raycast AI Commands, PromptLab)_
17. **Automatic on-device summaries on capture** `[v2]` - Long clips get a short summary generated automatically at capture time so the history list is scannable without opening each item. _(seen in: Recopy)_
18. **Structured-data extraction + smart field mapping on paste** `[future]` - Reads structured data from a copied form/table/document, identifies labels, and auto-inserts each value into the matching destination fields, remembering the mapping for next time. _(seen in: UiPath Clipboard AI)_
19. **AI smart collections / auto-grouping of related clips** `[future]` - AI clusters related clipboard items into collections or pinboards automatically rather than relying on manual folders. _(seen in: ClipMacs, Paste (pinboards))_
20. **Local vs cloud AI routing with provider choice** `[v2]` - Runs AI features on-device (Apple Intelligence / Ollama) by default with an option to route to cloud providers (OpenAI/Anthropic/Gemini) for heavier tasks. _(seen in: NanoClip, ClipMacs, Recopy)_

## Developer-focused features

21. **Snippet dynamic placeholders with chained modifiers** `[v1]` - Snippets expand with live placeholders ({cursor}, {date +2d}, {clipboard}, {argument}) whose values can be transformed inline via chained modifiers like {clipboard | trim | uppercase}. _(seen in: Raycast, NanoClip)_
22. **Clipboard-history offset access in snippets/placeholders** `[v2]` - Placeholders and snippet variables can pull not just the latest clip but an older history entry by index/offset (e.g. {clipboard offset}), so templates can reference several recently copied values at once. _(seen in: Raycast)_
23. **Form-field / fill-in templates** `[v1]` - A saved clip is a template with labeled input fields the user fills in at paste time, rendering the final text from those values. _(seen in: PasteBar, NanoClip, Raycast (argument fields))_
24. **JSON key-quote toggle (JSON keys ↔ JS object notation)** `[v2]` - One action switches between quoted JSON keys and unquoted JavaScript object-literal keys, beyond plain pretty-printing. _(seen in: ClipBear)_
25. **JSON minify (compact) in addition to pretty-print** `[v1]` - Collapse JSON to a single compact line, the inverse of beautify, often paired with validation. _(seen in: copycopy, Clippy, ClipMaster)_
26. **XML ↔ JSON conversion** `[v2]` - Bidirectional structural conversion between XML and JSON, not just reformatting each in place. _(seen in: CopyShareNow)_
27. **Color format detection and conversion (hex/RGB/RGBA/HSL) with live swatch** `[v1]` - Detects a copied color value and converts between hex, RGB(A), and HSL(A) with a live color preview. _(seen in: ClipBear, NanoClip (via blocks))_
28. **Unix/ISO timestamp detection and conversion** `[v1]` - Recognizes 10/13-digit epoch and ISO-8601 datetime strings and converts between epoch and human-readable date both ways. _(seen in: ClipBear)_
29. **UUID / MongoDB ObjectID timestamp extraction** `[future]` - Identifies UUID version and extracts the embedded creation timestamp (UUID v1/v7, Mongo ObjectID). _(seen in: ClipBear)_
30. **Cron expression to human-readable schedule** `[future]` - Detects a 5-field cron string in a clip and renders it as a plain-English schedule. _(seen in: ClipBear)_
31. **JWT header/payload copy-out as pretty JSON** `[v1]` - Beyond decoding inline, lets you copy just the decoded header or just the payload as pretty-printed JSON; detects tokens with or without the Bearer prefix. _(seen in: ClipBear)_
32. **Hash generator with multiple algorithms (MD5/SHA-1/SHA-256/SHA-512)** `[v1]` - Generates several hash digests from a clip, selectable by algorithm and case, not just a single MD5. _(seen in: CopyShareNow, ClipClip (MD5 lower/UPPER))_
33. **Whitespace/line operations: trim, collapse spaces, remove blank lines, sort, dedupe, reverse, join** `[v1]` - A suite of line-level text operations (trim, collapse duplicate spaces, sort alphabetically, remove duplicate lines, reverse order, join lines, add line numbers) applied to a clip. _(seen in: ClipClip, ClipMaster, copycopy, NanoClip)_
34. **Strip HTML/RTF formatting and extract source code from rich clips** `[MVP]` - Removes rich formatting to plain text, and can extract the underlying source/code from RTF/HTML content (e.g. code copied from an IDE or web page). _(seen in: ClipClip, ClipboardFusion, copycopy)_
35. **Quote normalization (smart/curly ↔ straight, single ↔ double)** `[v1]` - Converts smart/curly quotes to straight quotes and swaps single/double quotes, which routinely breaks pasted code. _(seen in: ClipClip)_
36. **Rich-text/HTML to clean Markdown conversion on paste** `[v2]` - Converts copied rich text or HTML into clean Markdown (and the reverse, Markdown to rich text/HTML), useful for docs and pasting code-bearing content into chat. _(seen in: clipboard2markdown, ClipClip (Markdown to HTML), NanoApps)_
37. **User-scriptable transform commands (JS/C#) with on-copy/on-paste triggers** `[future]` - Lets developers write custom transform scripts that run automatically on copy or paste, or on demand, for arbitrary multi-step processing. _(seen in: CopyQ (JS), ClipboardFusion (C# macros))_
38. **Run a shell/bash command against a clip** `[v2]` - Pipes a clip through an arbitrary shell command and stores the result, turning the clipboard into a quick CLI scratchpad. _(seen in: NanoClip, CopyQ)_
39. **Paste Stack / sequential multi-clip paste** `[v2]` - Queue several clips and paste them one-by-one in order with repeated paste, useful for filling forms or moving many values between files. _(seen in: Paste, Raycast)_
40. **Calculate inline math expression in a clip** `[future]` - Evaluates a copied arithmetic/math expression and returns the computed result. _(seen in: ClipClip, Raycast (calculator-adjacent))_
41. **URL tracking-parameter stripping** `[v2]` - Removes tracking/query junk (utm_*, fbclid, etc.) from a copied URL, leaving a clean link. _(seen in: NanoClip)_
42. **OCR text extraction from copied images** `[v2]` - Runs OCR on a copied image/screenshot (e.g. a screenshot of code or a stack trace) to recover selectable text. _(seen in: ClipBear (Apple Vision), NanoClip, Pastery)_

## Niche apps & unique/obscure features

43. **JavaScript scripting/command engine with automatic triggers** `[future]` - A full JavaScript scripting API where every command-line option is a function, letting users write custom commands that run automatically on every clipboard change or via global shortcuts (e.g. strip web colors, fetch URL titles). _(seen in: CopyQ)_
44. **Per-tab/per-folder filesystem synchronization** `[v2]` - Point a tab or collection at a folder (Dropbox/Drive/any synced dir) so its items become real files on disk and existing files in that folder appear as items, giving sync and external editing for free. _(seen in: CopyQ)_
45. **Encrypted tabs / encrypted collections** `[v2]` - Designate specific tabs or item groups as encrypted (GPG-based) so sensitive clips are stored encrypted at rest while the rest of history stays plaintext. _(seen in: CopyQ, Clibor (startup-password encryption), ClipAngel)_
46. **Item tags with custom colored markers and HTML styling** `[v1]` - Attach small colored icon/text tags to items (configurable, can hold HTML and 'mark:'-prefixed display rules) to flag important clips, separate from notes or pinning. _(seen in: CopyQ)_
47. **Hidden-content mime flag for sensitive items** `[v1]` - A per-item flag (mimeHidden) that hides the clip's content in the GUI while still keeping it usable, so secrets aren't shoulder-surfable in the list. _(seen in: CopyQ, PasteBar (data masking))_
48. **FIFO/LIFO sequential paste of multiple clips** `[v1]` - Queue several copied items and paste them one-by-one in first-in-first-out or last-in-first-out order with repeated paste presses, instead of choosing each time. _(seen in: Clibor, Ditto (paste multiple), ArsClip (paste stack))_
49. **Set/fixed phrases organized into groups with memos** `[MVP]` - A separate library of permanent canned snippets (distinct from rolling history), sortable into groups and annotated with memos, opened by its own hotkey. _(seen in: Clibor (set phrases), ArsClip (permanent items), Ditto (sticky clips))_
50. **Form/template clips with fillable fields** `[v2]` - Save a clip as a template containing named input fields so pasting prompts for or merges per-use values, automating repetitive structured data entry. _(seen in: PasteBar, ArsClip (templates with [CLIPBRD]))_
51. **Web-request / web-scraper clips** `[future]` - A clip type that fetches data from a web page or API on demand and pastes the extracted result, turning the clipboard manager into a lightweight automation tool. _(seen in: PasteBar, CopyQ (scripted URL-title fetch))_
52. **Application-aware auto-switching clip groups** `[v2]` - Maintain multiple item groups bound to specific programs and switch the active group automatically based on the foreground application. _(seen in: ArsClip)_
53. **Encrypted LAN clipboard sync between machines (no cloud)** `[future]` - Share clipboard items peer-to-peer over the local network to named friend machines using a group password, encrypted in transit, with no external server. _(seen in: Ditto)_
54. **Special-paste transforms at paste time** `[v1]` - Right-click a clip and paste it transformed: plain-text-only, lowercase, capitalized, with current time appended, or other on-the-fly conversions. _(seen in: Ditto, CopyQ (scripted))_
55. **Group-copy: bundle clips behind one shortcut** `[v2]` - Group several individual clips together and assign a predefined shortcut that copies/pastes the whole bundle at once. _(seen in: Ditto)_
56. **Multi-format capture with paste-as-format picker** `[v1]` - Store every copy in all available formats simultaneously (text, RTF, HTML, image, Excel) and let the user pick the target format at paste time via a 'Paste as' list or number keys. _(seen in: Comfort Clipboard Pro, ClipAngel)_
57. **Password/sensitive-app exclusion list** `[MVP]` - Block clipboard capture from designated applications (e.g. password managers, login windows) so secrets copied there never enter history. _(seen in: ClipAngel, PasteBar (lock + masking), Ditto (community-known))_
58. **Rich per-item provenance metadata and filtering** `[v1]` - Record and filter by source application, window title, and URL for each clip, so history can be searched/sliced by where it came from. _(seen in: ClipAngel)_
59. **Auto-search Quick Paste window that filters as you type** `[MVP]` - A floating paste palette that begins incremental fuzzy search the moment you start typing, with no separate search-box focus step. _(seen in: PasteBar, Clibor (incremental search), Ditto)_
60. **Built-in item editor to fix/merge clips before pasting** `[v1]` - Double-click an item to open an in-app editor (plain or rich text) to correct typos, reformat, or combine multiple entries, optionally via an external editor. _(seen in: CopyQ, ClipAngel (edit/rename/compare))_
61. **Cloud sync via user's own Google Drive account** `[future]` - Sync clipboard history and favorites across devices by storing the data file in the user's personal Google Drive rather than a vendor-hosted backend. _(seen in: 1Clipboard)_
62. **Numbered quick-paste positions for top-N clips** `[v1]` - Dedicated hotkeys (Paste Position 1-10) that paste the Nth most recent clip directly without opening or scrolling the history list. _(seen in: Ditto, Comfort Clipboard Pro (0-9 keys))_

## Launcher / productivity ecosystem features

63. **Paste Sequentially / Paste Stack (FIFO queue paste)** `[v1]` - Select multiple history items (or accumulate copies) and paste them one after another in order, with optional delay between pastes, ideal for filling forms or moving a list between documents. _(seen in: Raycast (Paste Sequentially), Paste (Paste Stack), Alfred PasteFlow workflow)_
64. **Save history item as a reusable snippet** `[v1]` - A one-keystroke action inside the clipboard viewer that promotes a copied clip into a permanent, keyword-expandable snippet library. _(seen in: Raycast (Save as Snippet), Alfred (Cmd+S in clipboard viewer))_
65. **Send clipboard item to AI / quick-action menu** `[v2]` - Per-item actions that route the selected clip into downstream tools (AI chat, translate, summarize, run command) directly from the clipboard list instead of only pasting. _(seen in: Raycast (Send to AI Chat, action panel), PopClip (extension actions))_
66. **Dynamic placeholders with the clipboard as a variable** `[v2]` - Templates, links, and snippets can embed {clipboard} (and offsets like {clipboard:1}) so the current or Nth clip is injected at expansion/run time. _(seen in: Raycast (Snippets & Quicklinks placeholders), Alfred (workflow {clipboard:N}), Keyboard Maestro (clipboard tokens))_
67. **Placeholder transform pipes / filters on a clip** `[v2]` - Inline modifiers transform a clip when inserted or run: uppercase, percent-encode, json-stringify, strip styles to plain text, etc. via a {clipboard | uppercase} style syntax or a Filter Clipboard action. _(seen in: Raycast (placeholder modifiers), Keyboard Maestro (Filter Clipboard))_
68. **Clipboard offset access (paste the Nth previous item)** `[v1]` - Expose not just the latest clip but item N-back as an addressable value for hotkeys, links, and automations (clipboard:0, clipboard:1, ...). _(seen in: Raycast (offsets), Alfred ({clipboard:N}))_
69. **Paste-as / format switching from history** `[MVP]` - Keep every original format that was copied (rich text, RTF, HTML, plain) and let the user choose which representation to paste per item, including paste-as-plain-text. _(seen in: Raycast (Paste as...), PopClip (Paste as Plain Text / Match Style), Maccy (paste-as-plain-text shortcut))_
70. **Merge / append copies (Cmd+C+C accumulation)** `[v2]` - A modifier gesture appends a newly copied selection onto the previously copied clip, building one combined clip from several copies without an intermediate editor. _(seen in: Alfred (Merging, Cmd+C+C), Alfred workflows (merge latest clips))_
71. **Run shell script / Shortcut on the selected clip** `[future]` - Pass the clip (or current selection) into a user shell script or macOS Shortcut as input (e.g. POPCLIP_TEXT) and put the result back, enabling arbitrary transformation actions. _(seen in: PopClip (Shell Script actions, Run Shortcut), Alfred (Run Script / Bypass for Shortcuts), Keyboard Maestro (Execute Script with clipboard))_
72. **Selection popup / point-of-cursor action bar** `[future]` - When text is selected, a floating menu of contextual actions (copy, paste, transform, send-to-app) appears at the cursor rather than requiring a hotkey to a separate window. _(seen in: PopClip (core behavior))_
73. **Paste-and-Enter / chained post-paste keystrokes** `[v2]` - An action that pastes and then automatically presses Return (or other keys), so a clip can be submitted in one step. _(seen in: PopClip (Paste and Enter extension))_
74. **Named clipboards (multiple persistent slots)** `[v2]` - Define any number of named slots you can copy into and paste from independently of the rolling history, as durable parking spots for frequently-swapped content. _(seen in: Keyboard Maestro (Named Clipboards + Clipboard Switcher))_
75. **Pinboards / collections of saved clips** `[v1]` - User-defined groups that hold a curated set of clips (links, snippets, images) organized by purpose, separate from the time-ordered history. _(seen in: Paste (Pinboards), Maccy (pinned items))_
76. **Shared / synced pinboards across devices and people** `[future]` - Pinboards and history sync across a user's Mac/iPhone/iPad and can be shared with collaborators who all see the same items in real time. _(seen in: Paste (shared pinboards, cross-device sync))_
77. **Pin item to top with assigned quick shortcut** `[v1]` - Pinning a history entry both keeps it at the top of the list and binds it to a persistent keyboard shortcut for instant recall. _(seen in: Maccy (Option+P pin with permanent shortcut), Raycast (pin))_
78. **Direct hotkey/alias to a specific clipboard slot (positions 0-5)** `[v1]` - Dedicated global shortcuts that paste a specific position in history without opening the picker, plus a custom alias/hotkey for the history command itself. _(seen in: Raycast (custom alias/hotkey; pastable-clipboard-history extension), Keyboard Maestro (Switcher hotkeys))_
79. **Per-app exclusion rules / password-manager auto-ignore** `[MVP]` - A rules tab to blocklist specific apps from being recorded and to automatically skip clips marked concealed/transient (passwords, 1Password, OTP). _(seen in: Paste (Rules), Maccy (blocklist + concealed/transient detection), Alfred (clipboard privacy settings))_
80. **Quick-action / type-ahead picker with fuzzy search and per-action keys** `[MVP]` - A keyboard-first picker where every operation (paste, paste-plain, pin, delete, edit) has its own shortcut and fuzzy/type-ahead search filters thousands of entries instantly. _(seen in: Maccy (fuzzy search, per-action shortcuts), Keyboard Maestro (Switcher type-ahead + filter), Raycast (action panel))_
81. **Grouped multi-copy as a single entry** `[v2]` - When several files or pieces of content are copied at once, they are stored as one grouped clipboard entry (and can be renamed) rather than split into separate items. _(seen in: Raycast (grouped clipboard entry, rename))_
82. **Bulk delete by time window / clear history range** `[v1]` - Actions to delete entries within a chosen time span (last hour/day) or wipe history quickly, beyond single-item deletion. _(seen in: Raycast (bulk-delete by time window))_

## Mobile & cross-device features

83. **Send clip to a specific device (directed push)** `[v1]` - Instead of broadcasting to all paired devices, the user picks one target device and pushes a single clip to it. _(seen in: KDE Connect, Pushbullet, Clipt, CrossPaste)_
84. **QR-code pairing of devices** `[v1]` - A device shows a QR code (or short authentication string) that another device scans to establish a trusted pairing without typing accounts or IPs. _(seen in: CrossPaste, Cross-Platform Clipboard Sync (extension), Clipt)_
85. **QR-code handoff of a single clip** `[v2]` - Render the current clipboard item as a QR code on screen so any camera-equipped device can grab it without being paired at all. _(seen in: CopyShareNow, Cross-Platform Clipboard Sync (extension), various QR-clipboard utilities)_
86. **Web fallback / no-install pickup link** `[future]` - A short code or URL lets a device with no app installed open a web page and receive the clipboard text via a browser. _(seen in: CopyShareNow, goonlinetools Online Clipboard, ClipShare)_
87. **LAN-only serverless sync (no cloud)** `[v1]` - Clipboard syncs directly over the local network between devices with no data ever passing through a vendor server. _(seen in: CrossPaste, KDE Connect, ClipSync, UniClipboard)_
88. **End-to-end encryption of synced clips** `[v1]` - Clipboard payloads are encrypted on-device so only paired devices (not any relay/cloud) can read them. _(seen in: CrossPaste, Microsoft Link to Windows clipboard, UniClipboard)_
89. **Sync of rich/typed content, not just plain text** `[v1]` - Cross-device transfer preserves images, files, HTML, rich text, URLs, and even colors as distinct typed clips rather than flattening to text. _(seen in: Apple Universal Clipboard, CrossPaste, KDE Connect)_
90. **Cross-device file/large-object transfer alongside clipboard** `[v2]` - Copying a file on one device makes it pasteable as the actual file on another, reusing the same send path as text clips. _(seen in: Apple Universal Clipboard, KDE Connect (Send Files), CrossPaste)_
91. **Latest-clip injection into the mobile keyboard** `[future]` - Synced PC/desktop clips surface as suggestions directly in the phone's on-screen keyboard so the user pastes without leaving the text field. _(seen in: Microsoft SwiftKey Cloud Clipboard, Microsoft Link to Windows + Gboard)_
92. **Third-party keyboard extension for full history on iOS** `[v2]` - A custom keyboard exposes the entire clipboard history inside any app, working around iOS limits on pasting from a separate app. _(seen in: Paste, PastePal, Clipboard - Paste Keyboard)_
93. **Share-sheet / share-extension capture on mobile** `[MVP]` - Because iOS/Android block background clipboard monitoring, the app registers in the system share sheet so users explicitly push content into history. _(seen in: Paste, PastePal, Clip (AltStore), QuickPaste)_
94. **Background clipboard capture workaround on iOS** `[future]` - Uses an allowed background entitlement (e.g. location services) to keep running and notify the user each time something new is copied, sidestepping iOS's no-background-clipboard rule. _(seen in: Clip (AltStore))_
95. **Home Screen / Control Center widgets and interactive toggles** `[v2]` - Widgets let users add a clip to history, paste a pinned item, or toggle sync from the home screen or Control Center without opening the app. _(seen in: Paste, PastePal, iOS clipboard managers generally)_
96. **Shortcuts / automation actions for clips** `[v2]` - Exposes clipboard actions (save clip, fetch item, push to device) to the OS automation layer so users wire them into Shortcuts or the Action Button. _(seen in: Paste, iOS Shortcuts-based clipboard managers)_
97. **Cross-device pinboards / synced pinned collections** `[v1]` - Pinned or grouped clips (e.g. task-specific pinboards) sync across devices, not just the raw rolling history. _(seen in: Paste, PastePal)_
98. **Auto-skip sensitive content on sync (password/transient exclusion)** `[v1]` - Rules detect passwords or content marked transient/sensitive and exclude it from history and cross-device sync. _(seen in: Paste, Apple (concealed/transient pasteboard types))_
99. **Browser extension as the desktop sync endpoint** `[v2]` - A Chrome/Edge/Firefox extension acts as the copy/paste bridge so users on machines where they can't install a native app still join the clipboard network. _(seen in: Pushbullet, ClipShare, Cross-Platform Clipboard Sync)_
100. **Proximity/same-account auto-handoff with no explicit send** `[v2]` - Recently copied content is automatically available to paste on a nearby trusted device for a short window, with no button press or device picker. _(seen in: Apple Universal Clipboard, Paste (handoff mid-task))_
101. **OCR/image text search across synced image clips** `[future]` - Copied images are run through on-device OCR so their text becomes searchable in the synced history. _(seen in: Paste (Apple Intelligence integration))_
102. **MITM-resistant pairing confirmation (SAS / compare-code)** `[v2]` - During pairing both devices display a short string the user verifies match, defeating man-in-the-middle attacks on the sync channel. _(seen in: CrossPaste (Short Authentication String))_
