# vbuff - Ideas and Improvements Backlog, Items 301-400

This file covers ideas 301-400 of the 500-idea backlog. Companion lists: engineering ideas (1-113) live in [../architecture.md](../architecture.md), product/strategy ideas (114-197) live in [../recommendation.md](../recommendation.md), user-facing/operations ideas (198-300) live in [ideas-top-300.md](ideas-top-300.md), and review backlog items (401-500) live in [ideas-401-500.md](ideas-401-500.md). ## Privacy and Trust Controls

301. **Capture consent prompt per sensitive source** `[M]` - When a source app first matches a sensitive profile, ask whether to skip, capture ephemeral, or capture normally. _Value: Gives users explicit control at the riskiest boundary._
302. **Secret-type specific handling** `[M]` - Treat OTPs, cards, API keys, SSH keys, and recovery codes with distinct TTLs and display masks. _Value: Avoids one blunt "secret" policy for very different workflows._
303. **Local privacy score** `[S]` - Show a simple score derived from encryption, denied sources, retention, sync settings, and telemetry state. _Value: Makes privacy posture understandable without reading settings._
304. **Capture rule simulator** `[M]` - Let users paste sample content into settings and see which rule would capture, mask, skip, or expire it. _Value: Makes regex and app rules testable before they affect real clips._
305. **Sensitive preview watermark** `[S]` - Overlay masked sensitive rows with the reason they were masked. _Value: Users understand whether the trigger was app, regex, entropy, or OS hint._
306. **Policy export for security review** `[S]` - Export capture policy, retention, and sync settings as redacted JSON. _Value: Helps teams approve the tool without exposing clip content._
307. **Clipboard hijack quarantine** `[M]` - If the clipboard changes between selection and paste-back, block paste and offer to inspect the unexpected new value. _Value: Defends against clipboard-replacement malware at the moment of harm._
308. **Known-secret denylist updater** `[M]` - Ship signed updates for secret detectors and password-manager source identifiers independent of app releases. _Value: Lets protection improve quickly without a full binary update._
309. **Per-rule audit counters** `[S]` - Show how often each skip/mask/expiry rule has fired. _Value: Helps users tune noisy rules and prove protections are active._
310. **Strict local-only workspace** `[M]` - A workspace profile where sync, export, webhooks, plugins, and network actions are impossible. _Value: Gives high-risk users a hard privacy boundary._
311. **Clip access reason prompt** `[L]` - Require plugins and integrations to include a reason string when requesting a clip. _Value: Creates an audit trail that is human-readable, not just technical._
312. **Encrypted crash-safe panic path** `[M]` - On panic, write only redacted state and encrypted diagnostic counters. _Value: Keeps debugging possible without dumping clipboard content._
313. **Per-device unlock timeout** `[S]` - Let each device lock history after a different idle period. _Value: A desktop and a travel laptop should not share the same risk settings._
314. **Private browsing source bridge** `[M]` - Browser extensions explicitly mark private-window copies as never-store by default. _Value: Restores privacy context the OS clipboard usually loses._
315. **Ephemeral clip visual countdown** `[S]` - Show a visible countdown on clips scheduled to self-destruct. _Value: Reduces surprise when temporary content disappears._

## Search and Recall

316. **Natural query parser** `[M]` - Parse queries like "urls from chrome last week" into structured filters. _Value: Makes advanced search usable without memorized syntax._
317. **Search result explanation** `[S]` - Show why a result matched: text, app, tag, source host, OCR, or semantic rank. _Value: Builds trust in ranked results._
318. **Search macros** `[S]` - Save common searches as named shortcuts, such as `work links` or `secrets today`. _Value: Turns repeated filters into reusable commands._
319. **Time-relative filters** `[S]` - Support filters such as `today`, `yesterday`, `last 2h`, and `before lunch`. _Value: Clipboard recall is often temporal, not textual._
320. **Pinned-result aliases** `[M]` - Give a pinned clip one or more short aliases that rank above raw text matches. _Value: Makes reusable snippets findable by intent._
321. **Typo-tolerant exact search** `[M]` - Add lightweight edit-distance correction for short queries over recent clips. _Value: Prevents small typos from hiding the desired row._
322. **App-aware default ranking** `[M]` - Boost clips previously pasted into the current foreground app. _Value: Makes recall feel context-aware without requiring filters._
323. **Query history with privacy filter** `[S]` - Store recent searches except those that look secret-like. _Value: Speeds repeated searches without leaking sensitive queries._
324. **Search scope lock** `[S]` - Temporarily lock search to a collection, app, or device until cleared. _Value: Reduces repeated narrowing during focused work._
325. **Result pinning within search** `[S]` - Pin a row to the top of the current query only. _Value: Supports temporary workflows without global pin clutter._
326. **Clip relationship graph** `[L]` - Link clips that were copied, transformed, pasted, or synced as part of the same chain. _Value: Helps users rediscover related material._
327. **Find source action** `[M]` - Reopen the source URL, file, or app when provenance exists. _Value: Makes the clipboard a bridge back to original context._
328. **Search miss feedback** `[S]` - If a query returns nothing, offer to broaden time, include archived clips, or search all devices. _Value: Makes empty states actionable._
329. **Compact query language help** `[S]` - Inline completion for `app:`, `kind:`, `tag:`, `device:`, `before:`, and `after:`. _Value: Teaches power search while typing._
330. **Audio-free paste confirmation** `[S]` - A subtle visual flash confirms paste success without sound or notification spam. _Value: Gives confidence without interrupting flow._

## Data Model and Storage

331. **Archive tier separate from history** `[M]` - Move old but kept clips into an archive that is excluded from normal popup search unless requested. _Value: Keeps daily recall fast while preserving long-term memory._
332. **Collection-level retention defaults** `[M]` - Let each collection define its own count, age, and size caps. _Value: Snippets, screenshots, and secrets deserve different lifetimes._
333. **Attachment manifest per clip** `[M]` - Store a manifest describing each flavor, thumbnail, blob, OCR text, and derived index. _Value: Makes repair and export deterministic._
334. **Blob integrity scrubber** `[M]` - Periodically verify CAS hashes and quarantine damaged blobs. _Value: Detects disk or sync corruption before paste fails._
335. **Open export schema versioning** `[S]` - Publish JSON and SQLite export schema versions with compatibility notes. _Value: Keeps user data portable across releases._
336. **Partial restore wizard** `[M]` - Restore selected collections, tags, or devices from backup instead of all-or-nothing import. _Value: Recovery should not overwrite good current data._
337. **Store compaction forecast** `[S]` - Estimate how much space VACUUM, blob GC, or thumbnail pruning would recover. _Value: Lets users decide whether maintenance is worth it._
338. **Sensitive data residency flag** `[M]` - Track whether a clip ever hit disk, stayed memory-only, synced, or was exported. _Value: Makes privacy guarantees auditable per item._
339. **Per-flavor paste preference** `[M]` - Remember that a clip should paste as plain text, HTML, image, or file list by default. _Value: Avoids repeated representation choice._
340. **Immutable clip core plus mutable annotations** `[M]` - Keep payload bytes immutable while tags, notes, pins, and collections live in separate mutable records. _Value: Prevents organization edits from changing canonical content._
341. **Garbage-collection dry run** `[S]` - Show which clips and blobs would be removed before running a cleanup. _Value: Avoids surprise data loss._
342. **Backup freshness badge** `[S]` - Show when the last verified backup/export happened. _Value: Makes durability visible._
343. **History import quarantine** `[M]` - Imported clips land in a review collection before joining normal history. _Value: Prevents noisy or unsafe legacy data from polluting recall._
344. **Schema downgrade export** `[L]` - Export data into an older schema when possible. _Value: Gives users an escape hatch after upgrades._
345. **Per-clip legal hold** `[M]` - Mark clips exempt from retention until explicitly released. _Value: Useful for support, audit, and incident-response workflows._

## Platform Fit

346. **macOS menu-bar status variants** `[S]` - Change icon/title subtly for active, paused, degraded, and locked states. _Value: The user can see capture health without opening the menu._
347. **Windows notification-area quick menu** `[S]` - Match the same actions in the Windows tray with platform-native labels. _Value: Keeps behavior consistent across OSes._
348. **Linux StatusNotifier fallback ladder** `[M]` - Try AppIndicator/SNI, then legacy tray, then a small always-available popup command. _Value: Linux desktop environments vary too much for one tray path._
349. **Wayland portal capability probe UI** `[M]` - Show exact portal availability for hotkeys, capture, paste, and foreground app identity. _Value: Turns Wayland ambiguity into inspectable state._
350. **Accessibility permission self-check** `[M]` - On macOS and Windows, test paste injection permission and surface copy-only fallback immediately. _Value: Avoids silent paste-back failure._
351. **Global hotkey conflict resolver** `[M]` - Detect unavailable shortcuts and offer nearby alternatives. _Value: First-run setup succeeds instead of failing quietly._
352. **Keyboard layout aware accelerators** `[M]` - Store shortcuts by physical key and display by current layout. _Value: Keeps shortcuts stable across QWERTY, AZERTY, and non-Latin layouts._
353. **Per-compositor Linux notes** `[S]` - Bundle concise guidance for GNOME, KDE, Sway, Hyprland, and X11. _Value: Users get relevant caveats instead of generic Linux warnings._
354. **Trayless mode** `[S]` - Run with hotkey only and no tray/menu-bar icon. _Value: Supports minimalists and environments where tray APIs are broken._
355. **Menu-only mode** `[S]` - Disable global hotkey and use menu/tray actions only. _Value: Useful on locked-down systems where hotkey registration is disallowed._
356. **Portable app profile** `[M]` - Let the app run with config/data beside the binary. _Value: Helps USB, test, and isolated work setups._
357. **Managed install policy file** `[M]` - Read organization defaults from a machine-wide policy file. _Value: Supports enterprise deployments without a cloud admin panel._
358. **Platform permission repair command** `[S]` - Add a `doctor` action that opens the right OS settings page or prints exact repair steps. _Value: Turns permission failures into guided recovery._
359. **Multiple display work-area tests** `[M]` - Add automated checks for notch, taskbar, dock, scaling, and mixed-DPI placement. _Value: Prevents popup placement regressions._
360. **Native theme synchronization** `[S]` - Follow OS light/dark/high-contrast changes live without restart. _Value: Makes vbuff feel resident and native._

## Collaboration and Teams

361. **Shared collection read receipts** `[L]` - Optional per-member read state for team snippets or shared clip packs. _Value: Teams can know whether critical snippets were seen without exposing private history._
362. **Snippet approval workflow** `[M]` - Draft, review, approve, and publish shared snippets. _Value: Prevents broken templates from reaching an entire team._
363. **Collection roles** `[M]` - Owner, editor, commenter, and viewer roles per shared collection. _Value: Makes sharing granular enough for real teams._
364. **Expiring shared clips** `[S]` - Shared links and team clips can expire automatically. _Value: Reduces lingering access to temporary material._
365. **Revocable external share link** `[M]` - Share one clip through an encrypted link that can be revoked. _Value: Provides safer ad-hoc sharing than chat-pasting secrets._
366. **Team default denylist** `[M]` - Organizations can ship a source-app and pattern denylist without seeing employee clips. _Value: Lets security teams enforce guardrails privately._
367. **Shared snippet variable catalog** `[M]` - Team snippets can reference organization-approved variables such as support phone, product names, or URLs. _Value: Reduces drift across templates._
368. **Per-team audit of config, not content** `[M]` - Admins see policy compliance and app health without clip payload access. _Value: Balances enterprise oversight with user privacy._
369. **Shared collection fork** `[S]` - Users can fork a shared collection into a private copy. _Value: Allows customization without breaking team source material._
370. **Conflict comments on shared snippets** `[S]` - Add lightweight comments to snippet edits and conflicts. _Value: Makes collaborative snippet maintenance less opaque._
371. **Emergency broadcast snippet** `[M]` - Push a high-priority snippet or notice to a team collection. _Value: Useful for incident response and support scripts._
372. **Team import validator** `[M]` - Validate shared snippet imports for placeholders, unsafe actions, and missing assets. _Value: Prevents bad bundles from spreading._
373. **Scoped team plugin approvals** `[L]` - Allow plugins only for specific shared collections or teams. _Value: Keeps extension power away from personal history._
374. **Shared collection changelog** `[S]` - Show who changed what metadata and when, excluding payload when privacy requires. _Value: Makes team libraries maintainable._
375. **Policy simulation for admins** `[M]` - Admins can test proposed rules against synthetic clips before rollout. _Value: Reduces accidental overblocking._

## Developer and Automation Surface

376. **Stable local RPC schema** `[M]` - Version every local API request/response with compatibility tests. _Value: Lets external tools depend on vbuff without breakage._
377. **CLI shell completion for filters** `[S]` - Complete tags, collections, kinds, and devices in shells. _Value: Makes the CLI feel polished for power users._
378. **Headless import mode** `[S]` - Import/export without launching the GUI or tray. _Value: Supports scripts, servers, and recovery workflows._
379. **Event stream replay cursor** `[M]` - Let clients resume `watch` from a cursor after disconnect. _Value: Makes automations reliable across restarts._
380. **Plugin test harness** `[M]` - Provide fake clips, permissions, and golden outputs for plugin authors. _Value: Encourages safe extension development._
381. **Signed action bundles** `[M]` - Package transform/action chains with signatures and declared permissions. _Value: Makes sharing automations less risky._
382. **Sandboxed network fetch action** `[L]` - A capability-scoped action can fetch from approved hosts only. _Value: Supports useful workflows without general network access._
383. **Local-only webhook receiver** `[M]` - Receive local events from trusted tools through loopback with scoped tokens. _Value: Integrates with editors and browsers without cloud dependencies._
384. **Scriptable backup command** `[S]` - A CLI verb exports encrypted backup plus manifest in one command. _Value: Makes scheduled backups easy._
385. **Machine-readable health endpoint** `[S]` - Expose status, capture health, DB size, and sync queue as JSON. _Value: Lets users monitor a resident daemon._
386. **Golden fixture corpus command** `[M]` - Export selected clips as sanitized test fixtures. _Value: Helps reproduce parsing and rendering bugs without leaking content._
387. **Transform marketplace metadata** `[L]` - Define categories, permissions, examples, and compatibility for shared transforms. _Value: Enables an ecosystem without making core huge._
388. **Plugin panic containment** `[M]` - A plugin crash disables only that plugin and records a redacted report. _Value: Extensions should not kill capture or paste-back._
389. **Automation rate limits** `[S]` - Per-token limits for reads, writes, and paste commands. _Value: Prevents runaway scripts from flooding history._
390. **Dry-run mode for every mutating API** `[S]` - CLI/RPC mutations can report intended changes without applying them. _Value: Makes automation safer and testable._
