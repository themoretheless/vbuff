# vbuff - Review Backlog, Items 401-500

This file covers review items 401-500 of the unified 600-item backlog. Companion lists: engineering ideas (1-113) live in [../architecture.md](../architecture.md), product/strategy ideas (114-197) live in [../recommendation.md](../recommendation.md), user-facing/operations ideas (198-300) live in [ideas-top-300.md](ideas-top-300.md), extended platform/product ideas (301-400) live in [ideas-301-400.md](ideas-301-400.md), and evidence-backed ideas (501-600) live in [ideas-501-600.md](ideas-501-600.md). Effort tags are `S`/`M`/`L`.

The tone here is intentionally review-like: proposals, improvements, problems, bugs, weak design choices, and places where the current implementation is still too coarse. The backlog is source material, not an automatic commitment. [../plan.md](../plan.md) remains the execution order; an idea only becomes planned work when it is pulled into a milestone.

## Current Implementation Problems

401. **Root binary owns too many reasons to change** `[M]` - `src/main.rs` currently wires config, capture, store refresh, hotkey polling, tray actions, GUI loop, paste sequencing, and tracing. _Value: Splitting those responsibilities makes the project readable and keeps small changes from touching the busiest file._
402. **Capture loop is polling-first instead of event-first** `[L]` - The MVP uses `arboard` polling everywhere, while the architecture needs native event backends. _Value: Calling this a known debt prevents the polling path from becoming the accidental platform abstraction._
403. **Capture gate is scattered and too shallow** `[M]` - Pause, whitespace skip, and app exclusion are inline in the capture thread, while future secret/concealed/retention rules need one ordered gate. _Value: One `CaptureDecision` pipeline is easier to test and harder to bypass._
404. **Store lock spans write, eviction, and full GUI reload** `[M]` - The capture thread holds the `Store` mutex while inserting, enforcing cap, and loading up to 1000 rows. _Value: Separating writer commands from read snapshots reduces UI jitter and deadlock risk._
405. **GUI snapshot refresh reloads too much** `[M]` - Every insert can reload the whole recent history slice. _Value: Incremental store events or keyset paging make large histories feel light._
406. **Self-write suppression is only last-hash local memory** `[M]` - The capture loop avoids immediate duplicate hashes but does not carry a typed self-write ledger. _Value: A paste-back can still echo through another manager or OS sync path._
407. **No explicit capture-health state machine** `[M]` - The app can log backend errors, but the UI does not own a structured `Capturing/Paused/Degraded/Unsupported` state. _Value: Silent loss is the worst clipboard-manager failure; health must be visible._
408. **Tray and popup share actions informally** `[S]` - Menu actions and popup actions are similar but not modeled as one command enum. _Value: A shared command model keeps keyboard, tray, CLI, and future IPC behavior consistent._
409. **Paste sequencing is frame-timer based** `[M]` - `PendingPaste` delays a keystroke by time rather than by explicit focus/popup lifecycle confirmation. _Value: Paste-back reliability should depend on target focus state, not elapsed milliseconds._
410. **Config is policy but not yet versioned** `[S]` - TOML settings can evolve without a config schema/version. _Value: Versioned config avoids silent default changes and supports migrations._
411. **The MVP store keeps flavor JSON in one row** `[M]` - This is simple, but it postpones indexed per-flavor queries, CAS references, and partial repair. _Value: The docs should mark the JSON shape as transitional so future migrations are expected._
412. **No repository-level design tokens yet** `[S]` - The GUI can drift into ad hoc spacing, colors, row heights, and icon rules. _Value: A tiny design-token module keeps a polished app from becoming visually inconsistent._
413. **Image thumbnail cache is local to the popup view** `[M]` - Thumbnail lifetime and memory pressure are not a first-class cache policy. _Value: Moving thumbnail policy behind a small cache boundary prevents leaks and keeps scrolling predictable._
414. **Hotkey failure is only logged** `[S]` - If registration fails, the user may not see why the app feels dead. _Value: Hotkey failure needs a visible remediation path._
415. **Current README overpromises target behavior near current status** `[S]` - The status section clarifies MVP gaps, but feature-heavy sections can still read as shipped. _Value: Better shipped/target labeling protects trust._

## SOLID and DRY Refactor Slices

416. **Introduce `capture` module around a single responsibility** `[M]` - Move polling/subscription, gate evaluation, hash computation, and store command emission out of `main.rs` behind a `CaptureSupervisor`. _Value: Capture becomes one digestible unit with clear tests._
417. **Introduce `app_commands` as the only side-effect vocabulary** `[S]` - Model `Show`, `Hide`, `Paste`, `CopyLatest`, `ClearHistory`, `Pause`, `Resume`, and `Quit` as shared commands. _Value: UI, tray, hotkey, and CLI stop duplicating behavior names._
418. **Define `HistoryReadModel` for GUI snapshots** `[M]` - Give the GUI a read-only model with recent clips, counts, and search results instead of direct store locking. _Value: UI code stops knowing persistence details._
419. **Define `HistoryWriter` for serialized mutations** `[M]` - Funnel insert, pin, delete, clear, and import through one writer actor or command queue. _Value: SQLite stays single-writer without spreading mutex usage._
420. **Extract `PasteCoordinator`** `[M]` - Own focus capture, clipboard write, popup hide, paste inject, and failure reporting in one object. _Value: Paste-back is too risk-heavy to remain split across frame wiring._
421. **Keep `vbuff-core` pure by adding ports, not concrete deps** `[S]` - When adding search, transforms, gates, or snippets, expose pure functions plus tiny traits. _Value: The crown-jewel tests remain fast and headless._
422. **Move UI row rendering into composable row components** `[S]` - Split search box, result row, badges, thumbnail, and action rail into small view functions. _Value: Designers and implementers can reason about one visual element at a time._
423. **Separate `ViewState` from `AppState`** `[M]` - Keep query, selected row, thumbnails, and focus timing separate from shared clip data and capture status. _Value: Prevents UI-only state from leaking into daemon/state concerns._
424. **Create a `CapturePolicy` object from config** `[M]` - Compile TOML config into a small immutable policy used by the capture gate. _Value: Validation happens once, and the gate gets a clean dependency._
425. **Make retention policy pure and store-agnostic** `[S]` - Compute victims in `vbuff-core`, execute deletes in `vbuff-store`. _Value: Eviction rules become easy to property-test._
426. **Use typed IDs for commands and store events** `[S]` - Avoid passing raw strings for clip ids, source apps, devices, collections, and command ids. _Value: The compiler catches category mistakes._
427. **Build platform fallback ladders behind one trait method each** `[M]` - A backend should report capability and chosen fallback without the GUI branching on OS internals. _Value: DRY platform behavior and honest UI badges come from the same source._
428. **Add a `DesignSystem` module to `vbuff-gui`** `[S]` - Centralize spacing, row height, color roles, radius, typography scale, and badge styles. _Value: Visual polish becomes enforceable code, not taste memory._
429. **Create a `Diagnostics` boundary** `[S]` - Redacted logs, health state, counters, and doctor checks should share one diagnostics model. _Value: Support surfaces reuse the same facts instead of re-probing._
430. **Document each slice with a one-screen README note** `[S]` - Each crate or major module gets a short "owns / does not own / tests" block. _Value: The project becomes learnable in small passes._

## Testing and Quality Gaps

431. **Missing capture-gate table tests** `[S]` - Add cases for paused, whitespace, excluded app, concealed, secret-like, oversized, and unsupported source. _Value: Fail-closed behavior becomes visible before native backends land._
432. **Missing duplicate/self-write regression tests** `[M]` - Simulate copy, paste-back write, OS echo, and recopy after TTL. _Value: Prevents loops and unwanted duplicate rows._
433. **Missing store migration fixture tests** `[M]` - Keep small DB fixtures per schema and migrate them forward in CI. _Value: History cannot be wiped by an upgrade surprise._
434. **Missing config migration tests** `[S]` - Load old TOML versions, apply defaults, and assert user overrides survive. _Value: Settings are user data too._
435. **Missing GUI row snapshot tests** `[M]` - Render core row states: empty, no result, paused, sensitive, pinned, image, long text, RTL, high contrast. _Value: Visual regressions become reviewable artifacts._
436. **Missing keyboard navigation contract tests** `[M]` - Verify Up/Down, Home/End, Enter, Esc, quick-pick, and focus retention in the popup. _Value: The keyboard-first promise is behavioral, not decorative._
437. **Missing paste failure simulation** `[M]` - Mock focus restore failure, clipboard write failure, and injection denial. _Value: The UI should degrade clearly instead of pretending paste succeeded._
438. **Missing large-history performance fixtures** `[M]` - Seed 50k and 100k clips with mixed payload sizes and track search/open latency. _Value: Search-as-you-type needs a budget early._
439. **Missing image-memory pressure test** `[L]` - Scroll through many image clips and assert cache size/texture count stay bounded. _Value: Prevents the classic clipboard-manager image bloat problem._
440. **Missing markdown link and backlog continuity check in CI** `[S]` - The docs now depend on many cross-links and ordered lists. _Value: Cheap automation prevents broken documentation navigation._
441. **Missing dependency policy check** `[S]` - Add `cargo audit`/`cargo deny` as a first gate. _Value: A privacy tool cannot treat supply-chain risk as optional._
442. **Missing unsafe boundary audit** `[S]` - Decide which crates forbid unsafe and where platform unsafe wrappers live. _Value: Review effort focuses on the actual native edges._
443. **Missing redaction tests for logs** `[M]` - Feed secret-like clips through warnings/errors and assert payload bytes never appear. _Value: Logs must not become the leak._
444. **Missing platform capability fixtures** `[M]` - Mock macOS permissions, Windows hotkey conflicts, X11, wlroots, KDE, and GNOME Wayland capability matrices. _Value: UI badges can be tested without every OS at hand._
445. **Missing release smoke checklist** `[S]` - Build, run, open popup, copy, paste, tray action, config path, and history persistence should be a small scriptable pass. _Value: Release confidence should not depend on memory._

## Designer-Grade UX and Visual System

446. **Popup needs a clear information hierarchy** `[S]` - Search input, results, selected row, badges, and actions need predictable weight and spacing. _Value: Users should recognize the target row in a glance._
447. **Rows should reserve stable slots for icon, text, metadata, and badges** `[S]` - Dynamic badges or thumbnails must not resize the list. _Value: No visual jitter while typing or scrolling._
448. **The tray menu should mirror the same command names everywhere** `[S]` - Tray labels, keyboard palette, README usage, and future CLI verbs should share terminology. _Value: The product teaches one mental model._
449. **Sensitive clips need calm but unmistakable treatment** `[M]` - Masking, peek, countdown, and local-only badges should be visible without panic colors. _Value: Privacy state is understood without making the UI stressful._
450. **No marketing hero inside the app shell** `[S]` - The first screen should be the usable history/popup, not a welcome poster. _Value: A clipboard tool earns trust by being useful immediately._
451. **Use icon buttons for repeated tools** `[S]` - Pin, favorite, delete, copy, preview, and paste-as-plain should have consistent icon affordances with tooltips. _Value: Dense tools stay scannable without text clutter._
452. **Use text labels for destructive commands** `[S]` - Clear history, wipe secrets, revoke device, and reset sync need explicit labels and confirmation. _Value: Dangerous actions should never hide behind ambiguous icons._
453. **Add a compact/comfortable density system** `[M]` - Row height, thumbnail size, and metadata visibility should adapt to display size and user preference. _Value: The same UI works on a laptop and a large monitor._
454. **Design empty states around the next action** `[S]` - Empty history, no search results, paused capture, locked vault, and unsupported backend should each offer one relevant action. _Value: Empty space becomes guidance, not explanation text._
455. **Make capability badges visually consistent** `[S]` - Active, paused, degraded, locked, local-only, synced, and sensitive states need a shared badge language. _Value: Users learn the state system once._
456. **Keep the command palette as the overflow surface** `[M]` - Rare actions belong in a searchable command palette, not permanently on every row. _Value: Power stays available without clutter._
457. **Add high-contrast and forced-colors design checks** `[M]` - Verify every badge, row, focus ring, and selection state under high contrast. _Value: Accessibility is a design constraint, not a post-pass._
458. **Use source-app icons sparingly** `[M]` - Source app identity should help scan, but content kind remains the primary row identity. _Value: Prevents the list from becoming a noisy app-logo wall._
459. **Preview pane should be an inspection tool, not a second app** `[M]` - Long text, image, HTML, diff, and metadata preview should stay compact and dismissible. _Value: Users inspect before paste without losing recall speed._
460. **Design the first-run path around one successful paste** `[M]` - Ask for permissions, show the hotkey, capture one safe clip, and complete a paste-back. _Value: The onboarding success moment is behavioral, not explanatory._

## Privacy, Security, and Trust Problems

461. **Encryption at rest is target, not current implementation** `[M]` - README and docs must keep distinguishing current bundled SQLite from planned SQLCipher. _Value: Trust depends on not overstating shipped protection._
462. **Secret detection is not yet in the capture path** `[M]` - The current code skips whitespace and excluded apps but not API keys, OTPs, passwords, or concealed hints. _Value: The privacy promise needs a real gate before broader dogfood._
463. **Deleted clips may remain in SQLite pages/WAL** `[M]` - Secure delete and WAL cleanup are not implemented yet. _Value: "Delete" must eventually mean more than removing a row reference._
464. **Config and DB paths need cloud-folder warnings** `[S]` - Users can accidentally place history in synced folders later. _Value: Avoids raw DB sync corruption and privacy leaks._
465. **No unlock/lock model yet** `[L]` - The app has no idle lock, master password, biometric gate, or session lock behavior. _Value: Local encrypted history still needs runtime exposure controls._
466. **No redacted bug-report flow** `[M]` - Diagnostics can help support but must never ship clip contents accidentally. _Value: Privacy-safe support reduces maintainer friction._
467. **No plugin/integration permission model yet** `[L]` - Future integrations must not get ambient full-history access. _Value: Extensibility without capability control would violate the product promise._
468. **No audit trail for sensitive decisions** `[M]` - Captured/skipped/masked reasons are not recorded for user review. _Value: Users need proof that privacy rules fired._
469. **No anti-hijack check before paste-back** `[M]` - The app writes a clip then sends paste, but does not verify nothing unexpected replaced the clipboard. _Value: Prevents clipboard-swap attacks at the moment of harm._
470. **No platform permission posture summary** `[S]` - The app cannot yet show whether Accessibility, hotkey, keychain, or Wayland portal permissions are healthy. _Value: Honest capability status keeps users from assuming unavailable protections exist._
471. **Sync must not arrive before data contracts freeze** `[M]` - IDs, hashes, flavor mappings, and schema must stabilize before multi-device replication. _Value: Sync amplifies every data-model mistake._
472. **Imports need quarantine by default** `[M]` - Imported competitor history may contain secrets, junk, or corrupted entries. _Value: Migration should not pollute the main history blindly._
473. **Logs need structured redaction by type** `[M]` - Clip id, byte size, kind, and source are useful; payload and sensitive metadata are not. _Value: Debuggability and privacy can coexist._
474. **Backups need verification, not just export** `[M]` - An encrypted export is not trustworthy until a restore check proves it opens. _Value: Recovery workflows should be testable before disaster._
475. **Security claims need user-visible evidence** `[M]` - At-rest encryption, local-only mode, sync exclusion, and secret skips need status surfaces. _Value: vbuff's positioning depends on proof, not vibes._

## Governance, Review, and Roadmap Hygiene

491. **Three-pass doc review as a recurring ritual** `[S]` - Pass 1 checks structure, pass 2 checks design/SOLID consistency, pass 3 checks links/counts/status claims. _Value: Large docs stay coherent as they grow._
492. **Backlog intake requires owner and reason** `[S]` - New items should name the problem they solve and the subsystem they touch. _Value: Ideas remain traceable instead of decorative._
493. **Every 100 backlog items needs a pruning pass** `[S]` - Merge duplicates, demote fantasies, and promote urgent debts. _Value: A 600-item backlog should get sharper, not just larger._
494. **Review findings should become tests or docs** `[S]` - If a review finds a recurring risk, encode it as a test, checklist, or architecture rule. _Value: The same issue is not rediscovered forever._
495. **Do not let design polish wait until after feature-complete** `[M]` - Row layout, keyboard feel, accessibility, and empty states must be validated while the core loop is still small. _Value: Polish is cheaper before habits harden._
496. **Define "MVP trust bar" before adding sync** `[S]` - No sync work until local capture, paste, deletion, encryption, and health status meet agreed gates. _Value: Networked history should amplify a reliable product, not a shaky one._
497. **Keep experimental AI behind explicit labels** `[S]` - AI/OCR/semantic features should be marked off-by-default and later-phase wherever mentioned. _Value: Protects the privacy brand from accidental scope creep._
498. **Use dogfood bugs as priority multipliers** `[S]` - Any bug that caused a maintainer to lose a clip or paste wrong should outrank speculative features. _Value: Real pain beats imagined polish._
499. **Tie release notes to trust claims** `[S]` - Each release should say which privacy, reliability, or platform guarantee improved. _Value: Users see steady progress on the product's core promise._
500. **Backlog-to-plan promotion gate** `[S]` - An item can move from backlog to plan only with scope, owner, test strategy, and cut criteria. _Value: Keeps 500 ideas from becoming 500 obligations._
