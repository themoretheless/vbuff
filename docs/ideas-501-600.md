# vbuff - Evidence-Backed Backlog, Items 501-600

This file adds 100 proposals derived from the repository and primary-source review in [repositories-research-100.md](repositories-research-100.md). These are review inputs, not automatic roadmap commitments. Promotion still requires the gate in `plan.md`: scope, owner, acceptance test, cut criteria, and evidence that the idea is not already solved by a smaller item.

Effort tags are `S`, `M`, and `L`. Evidence ids resolve in the research catalog.

## Native Clipboard Correctness

501. **Bound delayed-render work by flavor and deadline** `[M]` - Give every promised Windows/X11 flavor an independent deadline, cancellation path, and byte budget so one blocked producer cannot stall the entire capture generation. _Value: Complete formats when possible, but preserve watcher liveness under hostile or broken owners. Evidence: GH-004, GH-010, S-26._
502. **Model Windows clipboard contention as a state machine** `[M]` - Replace generic retries with bounded randomized `OpenClipboard` attempts, owner/sequence revalidation, and a final typed loss reason. _Value: Contention becomes diagnosable without storing a generation read after ownership changed. Evidence: GH-004, GH-038, S-18, S-26._
503. **Record native versus OS-synthesized formats** `[S]` - Persist whether a flavor was offered by the source, synthesized by Windows, or derived by vbuff, and prefer the true source representation during fidelity checks. _Value: Equivalent-looking formats stop masquerading as independent originals. Evidence: GH-013, GH-038, S-18._
504. **Build a strict bounded CF_HTML parser** `[M]` - Validate byte offsets, header lengths, charset assumptions, and fragment bounds before accepting HTML; retain malformed fixtures for regression. _Value: A common rich-text format cannot trigger out-of-bounds reads or silently slice the wrong fragment. Evidence: GH-004, GH-092, GH-093._
505. **Quarantine virtual-file clipboard objects** `[L]` - Treat delayed file descriptors and file-content streams as untrusted imports with count, path, size, time, and network-materialization limits before committing metadata or bytes. _Value: Copying a virtual file cannot cause an invisible download or unbounded read. Evidence: GH-004, GH-065, S-18._
506. **Preserve pasteboard item boundaries** `[M]` - Represent a capture as ordered items containing ordered flavors rather than flattening every type into one bag. _Value: Multi-file, multi-URL, and mixed-object copies can round-trip without attaching one item's format to another. Evidence: GH-013, GH-039, S-19._
507. **Choose macOS flavors through UTType conformance** `[M]` - Rank types by declared conformance and exact source order, not string-name heuristics, while preserving unknown custom types. _Value: New and application-specific pasteboard types degrade predictably. Evidence: GH-002, GH-039, S-19._
508. **Handle pasteboard sequence reset and rollover explicitly** `[S]` - Detect a decreasing or implausibly reset `changeCount`, establish a new generation epoch, and avoid comparing it to stale pre-sleep state. _Value: Long-lived sessions and pasteboard-server restarts do not suppress future captures. Evidence: GH-002, GH-029, S-19._
509. **Implement X11 INCR as a quota-aware finite-state machine** `[L]` - Track owner, target, chunk count, cumulative bytes, idle timeout, and terminal zero chunk for every transfer. _Value: Large selections remain correct without accepting infinite or cross-owner streams. Evidence: GH-008, GH-011, GH-035._
510. **Verify X11 `SAVE_TARGETS` handoff before claiming durability** `[M]` - Confirm requested targets were materialized and selection ownership transferred before reporting that a clip survives source exit. _Value: The UI never promises persistence for an uncompleted clipboard-manager handoff. Evidence: GH-008, GH-009, GH-035._
511. **Model PRIMARY and CLIPBOARD as separate channels** `[M]` - Give each X11 selection its own generation, policy, source, retention defaults, and self-write ledger instead of merging notifications early. _Value: Intent and provenance survive even when both selections contain the same bytes. Evidence: GH-008, GH-011, GH-035._
512. **React to Wayland seat hotplug and removal** `[M]` - Register data-control devices per seat, tear them down on removal, and expose which seat produced a capture. _Value: Docking, KVMs, and multi-seat systems do not leave dead listeners or cross-seat history. Evidence: GH-010, GH-035, GH-036, S-20._
513. **Snapshot Wayland offers before protocol object death** `[M]` - Copy target metadata and start bounded reads while the offer is valid; never retain a raw protocol object beyond its callback lifetime. _Value: Async storage work cannot dereference a destroyed offer. Evidence: GH-010, GH-036, S-20._
514. **Bind portal restore tokens to their environment** `[S]` - Store desktop, portal backend, session type, app id, and capability version beside each token and discard it when any binding changes. _Value: A stale grant cannot be mistaken for a valid permission after a compositor or package transition. Evidence: GH-040, GH-041._
515. **Normalize RDP clipboard format ids by negotiated names** `[M]` - Map remote registered ids to stable names, attach bridge lineage, and suppress only proven echoes across reconnects. _Value: RDP remapping does not corrupt custom formats or create a sync loop. Evidence: GH-004, GH-038._
516. **Separate archival fidelity from safe preview rendering** `[M]` - Keep canonical bytes immutable, then generate a sanitized, size-bounded preview in an isolated path with its own hash and parser version. _Value: Security hardening cannot silently alter what is pasted back. Evidence: GH-013, GH-014, GH-044._
517. **Move risky decoders behind a process boundary** `[L]` - Parse RTF, complex HTML, images, and future documents in a low-privilege worker with CPU, memory, file, and wall-clock limits. _Value: A malformed clip can kill a preview worker without taking down capture or gaining ambient history access. Evidence: GH-044, GH-065, GH-092._
518. **Budget decoded images, not compressed bytes** `[M]` - Reject or downsample using dimensions, pixel count, frame count, ICC/profile size, and projected allocation before decoding. _Value: A tiny compressed image cannot become a multi-gigabyte resident allocation. Evidence: GH-014, GH-044, GH-092._
519. **Audit declared MIME against detected content** `[S]` - Store declaration, conservative sniff result, mismatch reason, and chosen decoder without rewriting the original flavor label. _Value: Mislabelled payloads become visible and cannot steer a dangerous parser silently. Evidence: GH-013, GH-065._
520. **Export topology-preserving synthetic fixtures** `[M]` - Replace payload bytes with deterministic safe equivalents while preserving item count, flavor order, sizes, delayed-render states, and generation timing. _Value: Native clipboard bugs become shareable tests without leaking a user's copied content. Evidence: GH-013, GH-090, GH-092._

## Search, Text, and Interaction

521. **Map normalized matches back to grapheme spans** `[M]` - Build an offset map from the search surrogate to original byte and grapheme ranges so highlights never split a combining sequence. _Value: Search can normalize aggressively while selection and display remain byte-faithful. Evidence: GH-042, GH-043, S-10, S-11._
522. **Keep canonical and search text as separate representations** `[S]` - Store untouched bytes plus a versioned NFC/NFKC-casefold surrogate selected by content kind; never paste the surrogate. _Value: Canonically equivalent text is findable without violating round-trip fidelity. Evidence: GH-042, S-10._
523. **Isolate every untrusted bidi field in the row layout** `[M]` - Apply directional isolation independently to preview, source app, timestamp, badges, and actions. _Value: One RTL or override-heavy clip cannot reorder neighboring metadata or controls. Evidence: GH-042, GH-043, S-09._
524. **Reveal invisible controls in code and identifier inspection** `[M]` - Offer a safe inspector that names bidi controls, zero-width characters, non-breaking spaces, and unusual line separators without modifying the clip. _Value: Users can see dangerous structure before pasting commands or identifiers. Evidence: GH-013, S-09, S-12._
525. **Index UTS #39 skeletons only for high-risk identifiers** `[M]` - Compute confusable skeletons for domains, account ids, wallet-like addresses, and code identifiers, but keep them out of ordinary prose ranking. _Value: Security detection improves without flooding multilingual text with false alarms. Evidence: GH-057, S-12._
526. **Classify mixed-script risk at token level** `[M]` - Flag unexpected script mixtures only inside identifier-like tokens and explain which code points triggered the signal. _Value: Legitimate multilingual sentences remain calm while spoofable ids get scrutiny. Evidence: GH-057, S-12._
527. **Add locale-independent case-fold fixtures** `[S]` - Cover Turkish dotted/dotless I, Greek sigma, German sharp S, combining marks, and non-Latin scripts in exact and fuzzy search. _Value: Search behavior does not depend on the machine locale or a lossy lowercase call. Evidence: GH-042, GH-049, S-10._
528. **Make tokenization language-tailorable** `[L]` - Define a tokenizer port with ICU/CLDR-backed candidates for CJK, Thai, and other scripts lacking whitespace boundaries, benchmarked against the default. _Value: Unicode support extends beyond rendering into actual recall quality. Evidence: GH-049, GH-050, S-11._
529. **Truncate and move the caret by grapheme cluster** `[M]` - Make preview ellipsis, hit testing, selection, and keyboard movement share one grapheme segmentation implementation. _Value: Emoji ZWJ sequences and combining text never display or copy as broken fragments. Evidence: GH-042, GH-043, S-11._
530. **Provide an explicit literal-query mode** `[S]` - When query syntax is ambiguous or invalid, keep results stable, show the parsed interpretation, and allow one action to search the exact characters. _Value: Power syntax cannot make punctuation-heavy code clips mysteriously disappear. Evidence: GH-018, GH-019, GH-025._
531. **Anchor selection to clip identity during reranking** `[M]` - If async search stages return later, preserve the selected clip and scroll anchor unless it no longer matches. _Value: A user pressing Enter cannot paste a different row because results reordered under the cursor. Evidence: GH-019, GH-031, GH-032._
532. **Use a staged retriever with deterministic tie breaks** `[M]` - Run exact/prefix checks first, then indexed retrieval and bounded fuzzy expansion, with clip id as the final stable tie breaker. _Value: Common queries stay fast and identical inputs produce identical order. Evidence: GH-049, GH-050, GH-051, S-13._
533. **Version every ranking model and feature weight** `[S]` - Record the rank version with benchmark output and require an explicit relevance-delta review before changing it. _Value: Search quality changes become reproducible instead of subjective tuning. Evidence: GH-049, GH-050, S-13._
534. **Build a synthetic relevance benchmark** `[M]` - Create multilingual, code, URL, typo, app, recency, pin, and duplicate tasks with expected order, without sampling real user history. _Value: Ranking can be measured in CI without creating a privacy dataset. Evidence: GH-049, GH-051, S-13._
535. **Slice search SLOs by workload** `[S]` - Report p50/p95/p99 by query length, script, candidate count, history size, and cold/warm cache instead of one average. _Value: A fast ASCII demo cannot hide slow real-world tails. Evidence: GH-049, GH-090._
536. **Set a result-motion budget while typing** `[M]` - Measure row displacement across adjacent keystrokes and fail UX fixtures when unchanged top matches jump without a score reason. _Value: The list feels controlled even when fuzzy ranking is sophisticated. Evidence: GH-019, GH-050._
537. **Define one semantic action contract for every row** `[M]` - Expose select, paste, copy, pin, preview, delete, and state through platform-neutral accessibility actions before adapting to AccessKit/UIA/NSAccessibility. _Value: Keyboard, screen reader, and automation clients invoke the same behavior. Evidence: GH-034, GH-086, GH-091, S-16._
538. **Coalesce accessibility events during rapid filtering** `[M]` - Announce query/result-count changes at a bounded cadence while sending selection changes immediately. _Value: Screen readers remain informative instead of narrating every intermediate frame. Evidence: GH-034, GH-086, S-16._
539. **Give sensitive rows a non-leaking speech policy** `[M]` - Announce kind, sensitivity, age, and available reveal action, but never content or source metadata until explicit user intent. _Value: Visual masking is not undone by assistive output. Evidence: GH-055, GH-086._
540. **Offer non-drag and dwell-friendly organization controls** `[M]` - Every reorder/board action gets buttons or keyboard commands, with a large-target mode that preserves layout stability. _Value: Collections remain usable with switch control, eye tracking, tremor, or touch. Evidence: GH-088, S-14, S-15._

## Security and Privacy Engineering

541. **Persist detector version and decision evidence** `[M]` - Record detector id/version, matched structure, confidence bucket, allowlist rule, and final action without storing the matched secret substring. _Value: Reclassification and support can reproduce why a clip was skipped or protected. Evidence: GH-057, GH-058, S-07._
542. **Gate detector releases on calibrated precision and recall** `[M]` - Maintain per-secret-type thresholds and block rollout when a version regresses the agreed false-positive or miss budget. _Value: Adding signatures cannot quietly make fail-closed capture unusable. Evidence: GH-057, GH-058, S-07._
543. **Use synthetic secret canaries throughout the pipeline** `[S]` - Generate safe provider-shaped tokens and assert they never appear in DB plaintext, logs, crash artifacts, sync frames, exports, or support bundles. _Value: One corpus tests the whole privacy boundary without using real credentials. Evidence: GH-057, GH-092, GH-099._
544. **Separate offline structural validation from network verification** `[M]` - Check length, alphabet, prefix, checksum, and local context by default; any provider call requires explicit opt-in and a redacted request preview. _Value: Secret detection never exfiltrates a suspected secret to prove it is real. Evidence: GH-057, GH-058._
545. **Lint dangerous paste structure before terminal injection** `[M]` - Detect hidden controls, bidi overrides, ANSI escapes, multiline shell payloads, carriage returns, and paste-boundary tricks while leaving ordinary paste unchanged. _Value: Risk is surfaced at the moment an opaque command could execute. Evidence: GH-055, GH-065, S-09, S-12._
546. **Render terminal paste as shell-aware tokens without execution** `[L]` - Parse supported shells in a sandbox, show command boundaries and redirections, and fall back to a literal escaped view on ambiguity. _Value: Preview explains structure without pretending to prove a command safe. Evidence: GH-017, GH-065._
547. **Never verify long addresses by prefix alone** `[S]` - High-risk previews must expose the full value, grouped checksum, and changed middle/suffix ranges; abbreviated display is forbidden in confirmation. _Value: The UI does not train the exact weak behavior exploited by clipboard meddling attacks. Evidence: S-06._
548. **Pin trusted recipient fingerprints locally** `[M]` - Let users assign a local label to a full normalized address or account id and warn on confusable or near-match substitutions. _Value: Repeated transfers gain a stronger comparison baseline than visual memory. Evidence: GH-055, S-06, S-12._
549. **Generate a platform-specific clipboard threat posture** `[M]` - Derive an in-app report from actual backend, permissions, native history, remote session, screen sharing, sync, and lock state. _Value: Privacy claims match the machine's observable capabilities rather than a generic badge. Evidence: GH-041, GH-055, GH-065, S-17._
550. **Mask previews while the screen is shared or recorded** `[M]` - Detect supported recording/share states, default sensitive rows to metadata-only, and let the user temporarily reveal with a visible timer. _Value: A local vault does not leak through routine demos and support calls. Evidence: GH-055, S-15._
551. **Scan crash artifacts for a privacy canary before release** `[M]` - Trigger controlled failures after processing synthetic secrets, then inspect panic files, minidumps, traces, and OS reports for payload fragments. _Value: Redaction is verified against actual artifacts, not only logger unit tests. Evidence: GH-090, GH-092, GH-099._
552. **Measure sensitive-buffer lifetime** `[L]` - Instrument debug builds to count plaintext copies, maximum live duration, and zeroization completion by operation without recording bytes. _Value: Memory-hardening work targets measured exposure rather than ritual `zeroize` calls. Evidence: GH-055, GH-061._
553. **Specify the vault-key lifecycle as a state machine** `[M]` - Define creation, unwrap, active, idle-lock, session-lock, rotation, recovery, and destruction transitions with forbidden operations in each state. _Value: Encryption, UI locking, and background capture stop making contradictory assumptions. Evidence: GH-055, GH-059, GH-064._
554. **Use per-record data keys for cryptographic erasure** `[L]` - Encrypt each payload with a random DEK, wrap it under the vault/collection key, and delete the wrapped DEK when a record is purged. _Value: Residual database pages contain unusable ciphertext after key-reference removal. Evidence: GH-060, GH-061, GH-064._
555. **Verify key-reference closure after purge** `[M]` - Walk live records, wrapped keys, backups, sync outbox, thumbnails, and exports to prove no active key can decrypt a deleted canary. _Value: A purge receipt reflects cryptographic reachability, not just SQL row count. Evidence: GH-064, GH-080._
556. **Parse untrusted imports in a disposable worker** `[L]` - Give import adapters read-only access to one file, no network, no vault key, bounded resources, and a typed sanitized output stream. _Value: A malicious competitor backup cannot inherit vbuff's ambient privileges. Evidence: GH-065, GH-092, GH-093._
557. **Preflight archives against traversal and expansion attacks** `[M]` - Reject absolute/parent paths, links, duplicate canonical names, excessive nesting, and decompression ratios before extraction. _Value: Import/export convenience cannot overwrite files or exhaust disk. Evidence: GH-065, GH-092._
558. **Issue user-visible receipts for plugin data access** `[M]` - Record which capability read which clip class, why, and whether bytes left the process, with payload always redacted. _Value: Extension permission is inspectable after consent, not a one-time dialog. Evidence: GH-017, GH-021, GH-059._
559. **Ship a compile-time no-network profile** `[M]` - Produce a local-only artifact with network/sync code absent, then run it under a socket-deny harness and publish the result. _Value: The strongest privacy mode is enforced by the binary, not a setting that could regress. Evidence: GH-030, GH-065._
560. **Evaluate security nudges by changed behavior** `[M]` - Instrument only local aggregate outcomes such as canceled dangerous paste or chosen safe alternative, and remove warnings that add friction without improving decisions. _Value: Security UX is judged by safer actions, not dialog count. Evidence: S-08._

## Local-First Sync and Transfer

561. **Write a formal replication operation algebra** `[M]` - Define create, add-flavor, annotate, pin, delete, expire, revoke, and compact operations with preconditions and merge semantics before transport code. _Value: Sync implements one reviewable datatype rather than scattered message handlers. Evidence: GH-072, GH-073, S-02, S-03._
562. **Exhaustively permute small replication histories** `[M]` - Generate short operation traces and test every reorder, duplication, delay, partition, crash, and reconnect for convergence and invariants. _Value: Counterexamples appear before they become irreproducible multi-device bugs. Evidence: GH-072, GH-073, GH-074, S-03._
563. **Compute a causal-stability frontier before compaction** `[L]` - Retain tombstones and superseded operations until every non-forgotten peer has acknowledged a covering version vector. _Value: Garbage collection cannot resurrect deleted clips on a late device. Evidence: GH-072, GH-074, S-03._
564. **Expose per-device acknowledgement vectors** `[M]` - Show last-seen operation frontier, pending metadata, pending bytes, and deletion coverage for each trusted device. _Value: 'Synced' becomes evidence rather than one global spinner. Evidence: GH-066, GH-072, GH-082._
565. **Make forgotten-device deletion semantics explicit** `[M]` - Require a deliberate forget action, record its epoch, explain that old backups need re-pairing, and let compaction advance only afterward. _Value: Offline devices do not block cleanup forever or silently rejoin with stale state. Evidence: GH-066, GH-073, GH-080._
566. **Join new peers through a snapshot-plus-tail barrier** `[L]` - Pin an operation frontier, send an authenticated snapshot at that frontier, then replay the tail before declaring the peer current. _Value: A busy history cannot lose operations between snapshot and live subscription. Evidence: GH-066, GH-074, GH-082._
567. **Use read-only compatibility on protocol mismatch** `[M]` - Negotiate schema, operations, formats, and crypto suites; when mutation compatibility is unsafe, allow status/upgrade guidance without applying writes. _Value: Mixed-version peers fail conservatively instead of corrupting shared state. Evidence: GH-066, GH-071, GH-082._
568. **Negotiate flavor capabilities independently** `[M]` - Exchange supported native/canonical formats and deterministic fallback rules before transferring payload bytes. _Value: A Linux peer and macOS peer agree which representation is authoritative without discarding unknown flavors. Evidence: GH-006, GH-068, GH-071._
569. **Sync metadata before lazy blob content** `[M]` - Replicate ids, hashes, kind, preview policy, tombstones, and availability first; fetch large encrypted blobs by priority or on demand. _Value: Lists converge quickly on slow links without pretending every image is local. Evidence: GH-066, GH-079, GH-080._
570. **Chunk large encrypted blobs by content-defined boundaries** `[L]` - Use a reviewed chunker with min/target/max sizes so resumed or slightly changed payloads reuse authenticated chunks. _Value: Large screenshots and files do not restart from byte zero or duplicate every byte. Evidence: GH-080, GH-081._
571. **Authenticate resumable chunks with a tree root** `[L]` - Bind chunk order, sizes, total length, and content to a manifest/Merkle root before exposing a completed flavor. _Value: Resume and dedup cannot splice valid chunks into the wrong payload. Evidence: GH-079, GH-080, GH-081._
572. **Forbid QUIC 0-RTT for mutating operations** `[S]` - Permit only explicitly idempotent capability/status requests before handshake confirmation; create/delete/pin/revoke frames wait for 1-RTT keys. _Value: Replay optimization cannot duplicate or resurrect clipboard operations. Evidence: GH-071, S-25._
573. **Bind SPAKE2 pairing to the full transcript** `[M]` - Include app id, protocol version, both device ids, chosen ciphersuite, capabilities, and human-visible names in context and key confirmation. _Value: A short code cannot be redirected into an unknown-key-share or downgrade flow. Evidence: GH-069, GH-070, S-24._
574. **Offer an accessible short-authentication string** `[M]` - Present the same verification as grouped words, digits, QR, copyable text, and screen-reader-friendly speech with explicit confirmation on both peers. _Value: Pairing trust does not depend on vision, color, or precise camera use. Evidence: GH-070, GH-086, GH-088._
575. **Pin device identity continuity after pairing** `[M]` - Warn on a changed long-term key even when the device name and network address match, and require an explicit replacement flow. _Value: Reinstall, restore, or impersonation cannot silently inherit an old trust label. Evidence: GH-066, GH-077, GH-078._
576. **Test revoked peers returning from long partitions** `[M]` - Simulate stale session tickets, cached discovery, restored backups, old snapshots, and relay queues after revocation. _Value: Revocation remains true under the hardest offline rejoin paths. Evidence: GH-066, GH-077, GH-080._
577. **Apply a privacy budget to local discovery** `[M]` - Rotate unauthenticated discovery nonces, minimize advertised metadata, rate-limit probes, and reveal stable identity only inside an authenticated pairing flow. _Value: Nearby convenience does not become a persistent device-tracking beacon. Evidence: GH-067, GH-071, S-05._
578. **Expose direct, relayed, offline, and blocked states distinctly** `[S]` - Show current path, last transition reason, expected privacy/cost tradeoff, and a retry action without network jargon. _Value: Peer-to-peer branding does not hide relay use or silent unreachability. Evidence: GH-066, GH-077, S-04._
579. **Separate backup epochs from sync epochs** `[L]` - A restore imports records into a new epoch and reconciles deletion knowledge before publishing operations to peers. _Value: Restoring an old backup cannot automatically resurrect data intentionally deleted everywhere. Evidence: GH-073, GH-080, GH-082._
580. **Support signed air-gapped transfer bundles** `[L]` - Export a bounded encrypted manifest plus chunks with sender epoch, recipient, expiry, and nonce; reject replayed or wrong-recipient imports. _Value: Isolated devices can exchange data without weakening identity and deletion semantics. Evidence: GH-060, GH-080, S-23._

## Verification, Release, and Evidence

581. **Pin and assert a fixed SQLite engine version** `[S]` - Keep the bundled engine on a fixed line (currently SQLite 3.53.2), expose `sqlite_version()` in diagnostics, and fail release CI if it falls into a denied range. _Value: Wrapper changes cannot hide a known storage-engine vulnerability. Evidence: GH-046, GH-048, S-21._
582. **Reproduce the WAL-reset race in a regression harness** `[L]` - Add a test-only synchronization hook that overlaps writer and checkpoint connections at the documented critical boundary, then verify committed rows survive. _Value: vbuff tests the exact concurrency mode behind a current corruption advisory. Evidence: GH-046, S-21._
583. **Inject torn and reordered filesystem writes** `[L]` - Run store transactions through a faulting VFS or equivalent shim that cuts sectors, reorders writes, crashes at every boundary, and validates all-or-nothing recovery. _Value: Crash safety becomes measured under realistic failure mechanics. Evidence: GH-046, GH-053, S-22._
584. **Test disk exhaustion at every storage layer** `[M]` - Fill space separately during DB growth, WAL checkpoint, temp sort, blob write, thumbnail generation, export, and migration rollback. _Value: One `disk full` path cannot leave a half-committed item or misleading success state. Evidence: GH-046, GH-080, GH-090._
585. **Classify unsafe live-database paths** `[S]` - Detect known cloud-sync, network, removable, and unsupported-locking locations, explain the risk, and keep the canonical store in local app data. _Value: WAL sidecars are never treated as ordinary syncable files. Evidence: GH-046, GH-066, S-21._
586. **Cross every migration with every interruption point** `[L]` - For each released schema fixture, crash before/after each DDL/data/checkpoint step, reopen, and require either old-valid or new-valid state plus preserved pins. _Value: Forward-only migration is safe under interruption, not only happy-path input. Evidence: GH-046, GH-047, GH-082._
587. **Model-test the store command state machine** `[M]` - Generate insert, dedup, pin, favorite, delete, clear, cap, crash, and reopen sequences and compare SQLite to a small pure reference model. _Value: Interaction bugs emerge from arbitrary sequences, not just hand-written examples. Evidence: GH-046, GH-090, GH-099._
588. **Differential-test search against a slow oracle** `[M]` - Compare optimized SQL/FTS/fuzzy results to a straightforward in-memory implementation over generated Unicode histories. _Value: Index and ranking optimizations cannot silently drop or invent matches. Evidence: GH-049, GH-050, GH-099, S-13._
589. **Fuzz normalization, segmentation, and highlight mapping together** `[M]` - Generate malformed UTF-8 boundaries where applicable, combining marks, bidi controls, ZWJ emoji, and long grapheme clusters, then assert valid mapped spans. _Value: Internationalized search fails safely under adversarial text. Evidence: GH-042, GH-092, GH-093, S-09, S-10, S-11._
590. **Seed fuzzers from sanitized native-format topology** `[M]` - Feed minimized Windows, macOS, X11, and Wayland fixture structures into parser fuzz targets while replacing user payload bytes. _Value: Coverage starts from realistic protocol states without checking secrets into the repository. Evidence: GH-013, GH-092, GH-093._
591. **Set a mutation-score floor for trust decisions** `[M]` - Require tests to kill mutations that invert capture, sync, redaction, retention, purge, and permission outcomes; report surviving mutants by owner. _Value: Branch coverage cannot pass when the safety decision itself is untested. Evidence: GH-098, GH-099._
592. **Assign dependency review criteria by risk tier** `[M]` - Native input, crypto, parsers, updater, network, and storage crates require stronger audits and narrower exemptions than pure utility crates. _Value: Supply-chain review effort follows blast radius. Evidence: GH-094, GH-095, GH-096._
593. **Review every SBOM delta as release evidence** `[S]` - Generate a dependency/component diff, call out new native code and licenses, link advisory status, and require an owner for exceptions. _Value: A signed SBOM becomes a review tool instead of a ceremonial artifact. Evidence: GH-094, GH-095, GH-097._
594. **Publish artifact-level size and startup deltas** `[S]` - Measure each packaged target's compressed/install size, dynamic libraries, cold start, and hotkey-ready time against the previous release. _Value: Framework or dependency growth cannot quietly erode the resident-tool budget. Evidence: GH-030, GH-031, GH-097._
595. **Run an install/uninstall residue canary scan** `[M]` - Use a synthetic secret, exercise history/cache/export/key storage, uninstall with wipe, then scan documented paths and sidecars for plaintext or live keys. _Value: Clean removal is demonstrated end to end. Evidence: GH-055, GH-090, GH-097._
596. **Snapshot native accessibility trees per platform** `[M]` - Store reviewed semantic trees for popup states and diff names, roles, values, actions, focus, and live regions in CI. _Value: Visual stability cannot hide an accessibility regression. Evidence: GH-034, GH-086, GH-091, S-16._
597. **Replay the full native workflow through semantic automation** `[L]` - On each OS, copy in a fixture app, summon the real global hotkey, filter, paste, use tray actions, relaunch, and verify persisted state. _Value: Unit-tested modules are proven to compose through real platform surfaces. Evidence: GH-089, GH-090, GH-091._
598. **Measure idle energy and wakeups on real hardware** `[L]` - Run overnight capture-idle and burst scenarios on representative laptops, recording wakeups, CPU, memory, thermal state, and battery delta by subsystem. _Value: 'Near zero idle cost' is validated beyond synthetic microbenchmarks. Evidence: GH-002, GH-029, GH-045._
