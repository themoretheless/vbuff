# Post-600 candidate ideas

These ten ideas were added on 2026-07-20 after batch 201-250 exposed concrete lifecycle, accessibility, and responsive-layout follow-ups. They are evidence-backed candidates, not an expansion of the canonical 1-600 execution goal and not implementation claims. Evidence ids resolve through [repositories-research-100.md](repositories-research-100.md).

601. **Retention impact preview over live metadata** `[M]` - Before applying a rule, show exact candidate count, aggregate bytes, protected rows, grace-key availability, and deletion class without decrypting clip previews. _Value: Per-kind cleanup becomes an inspectable decision instead of a destructive settings guess. Evidence: GH-055, GH-080, S-08._
602. **Normalization collision guardrail** `[M]` - Require a normalized digest plus bounded token/grapheme similarity before collapsing variants, and retain an audit sample of rejected groups without payload logs. _Value: Cosmetic dedup cannot hide semantically different Unicode text that happens to normalize alike. Evidence: GH-049, GH-099, S-10, S-11._
603. **Suggested-pin decay and dismissal memory** `[S]` - Rank reuse with an explainable half-life and remember local dismissals for a bounded cooldown rather than repeatedly nagging on raw copy count. _Value: Suggestions follow current habits and respect a user's explicit no. Evidence: GH-019, GH-050, S-13._
604. **Typed recovery-key availability state** `[M]` - Model unavailable, locked, active, rotating, and failed key-provider states and make every grace-bin mutation require `Active`. _Value: Encrypted Undo cannot silently fall back to plaintext or hard deletion during keychain outages. Evidence: GH-055, GH-059, GH-064._
605. **Derived-output lineage receipts** `[M]` - Record canonical hash, transform/pipeline versions, permission-grant id, output hash, and time while redacting input/output bytes. _Value: A pasted derivative can be reproduced and audited without mutating history or retaining another plaintext copy. Evidence: GH-017, GH-045, GH-059._
606. **DPR and viewport layout invariant generator** `[M]` - Generate desktop/compact sizes, fractional DPRs, font scaling, RTL/CJK samples, and long labels, then assert no overlap, clipping, target shrink, or breakpoint disagreement. _Value: Responsive regressions become deterministic tests instead of screenshot luck. Evidence: GH-026, GH-029, GH-042, GH-090, S-14, S-15._
607. **Accessible command-transition oracle** `[M]` - For each palette/row/settings command, verify semantic action, focus destination, state change, and live announcement as one transition, not isolated tree snapshots. _Value: A control can be named correctly yet still strand keyboard and screen-reader users after activation. Evidence: GH-034, GH-086, GH-091, S-16._
608. **Content-free clipboard workload synthesizer** `[M]` - Generate replayable traces of kind, size bucket, timing, duplicate relation, and user action while using synthetic payloads only. _Value: Burst capture, scrolling, dedup, retention, and recovery can share realistic deterministic performance scenarios without collecting user history. Evidence: GH-046, GH-090, GH-099._
609. **Pairwise cross-feature invariant gate** `[M]` - Generate interactions among pause, incognito, sensitive state, transform, sync eligibility, expiry, delete/undo, and crash/reopen, then compare against a pure policy model. _Value: Safety failures that require two individually correct features stop escaping single-feature tests. Evidence: GH-065, GH-090, GH-098, GH-099._
610. **Synthetic recovery drill** `[S]` - On explicit request, create a canary clip, spill it to CAS, encrypt-delete it, remove the live blob, restore it, verify the hash, and scrub the drill record. _Value: Users and release CI can prove that current key, database, CAS, and recovery paths compose before a real accident. Evidence: GH-055, GH-080, GH-090._
