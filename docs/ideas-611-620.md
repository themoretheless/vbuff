# Post-600 candidate ideas, pass 2

These ten ideas were added on 2026-07-21 after the final review of batch 251-300 exposed concrete state-machine, replay, ranking, configuration, and release-evidence follow-ups. They are evidence-backed candidates, not an expansion of the canonical 1-600 execution goal and not implementation claims. Evidence ids resolve through [repositories-research-100.md](repositories-research-100.md).

611. **Authenticated shared-snippet approval envelope** `[L]` - Sign proposal identity, content hash, base/proposed revisions, approver identity, decision, and expiry, then verify quorum against durable team membership before constructing approved state. _Value: A serialized or replayed approval cannot impersonate a reviewer or bypass the local constructor-controlled state machine. Evidence: GH-059, GH-062, GH-066, S-23._
612. **Monotonic replay watermark with validity horizon** `[M]` - Persist the highest accepted event id and the maximum observed validity horizon independently, and property-test mixed TTLs, clock skew, restart, and pruning. _Value: A newer short-lived event cannot shorten replay memory and make an older long-lived mutation valid again. Evidence: GH-062, GH-065, GH-099, S-25._
613. **Injected snapshot clock and timezone** `[M]` - Make every visual fixture consume an explicit clock, timezone, locale, and scale context instead of deriving age labels from wall time. _Value: Boundary-sensitive golden images remain deterministic across slow runners, midnight transitions, DST, and host configuration. Evidence: GH-029, GH-090, GH-098._
614. **Profile current-versus-target dry run** `[M]` - Before profile application, show field-level changes plus exact retention candidates, protected rows, estimated bytes, and restart-only effects from live metadata. _Value: A friendly preset cannot conceal cleanup impact or overwrite a carefully customized safety posture. Evidence: GH-017, GH-055, GH-090, S-08._
615. **Session-protection semantics matrix** `[M]` - Specify and execute one table covering explicit delete, clear-all, automatic clear, expiry, count eviction, lock, restart, grace-bin restore, and pinned state. _Value: Every destructive path gets one reviewable answer instead of inheriting whichever exception a nearby SQL query happened to use. Evidence: GH-055, GH-065, GH-090, GH-099._
616. **Source recency independent from presentation order** `[S]` - Derive recent-source filters from capture timestamps with deterministic tie-breaking, then apply pinning and UI ordering only after source selection. _Value: An old pinned item cannot displace the source of a genuinely recent clip or make filters disagree with history. Evidence: GH-019, GH-049, GH-050, S-13._
617. **AST-preserving settings writes end to end** `[L]` - Route every settings mutation through typed validation plus `toml_edit`, retaining comments and unknown forward-compatible keys while making atomic rollback evidence reusable. _Value: The migration command cannot preserve a user's hand-edited file while ordinary GUI or CLI saves later erase it. Evidence: GH-017, GH-046, GH-053._
618. **Self-verifying release evidence bundle** `[M]` - Emit a manifest that names and hashes binaries, checksums, SBOMs, provenance, source revision, tool versions, and partial failures, plus a command that verifies the downloaded bundle offline. _Value: An uploaded folder becomes independently replayable evidence rather than a workflow log claiming artifacts that may be absent. Evidence: GH-094, GH-095, GH-096, GH-097, GH-100._
619. **Invariant-safe integration constructors** `[M]` - Keep privileged transition state private and non-deserializable; accept wire DTOs only through validated constructors and mutation methods, with serde and mutation tests for forged terminal states. _Value: Public fields or derived deserialization cannot skip preview, approval, consent, or one-shot transitions. Evidence: GH-062, GH-065, GH-098, GH-099._
620. **Endpoint key rotation and revocation windows** `[L]` - Version webhook/MCP endpoint keys, accept a bounded overlap for in-flight reads, reject mutations under retired keys, persist revocation watermarks, and expose a content-free audit trail. _Value: Rotation can be routine without creating indefinite old-key access or breaking every legitimate in-flight request at once. Evidence: GH-059, GH-062, GH-064, S-23._
