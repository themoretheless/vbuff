# Implementation batch 301-350

Reviewed and accepted at the runtime/foundation/adapted/native-required level on 2026-07-22. This ledger is the execution overlay for backlog items 301-350 in [ideas-301-400.md](ideas-301-400.md). Acceptance here means the documented local batch checks pass; it does not waive the separate native, encryption, transport, or release gates listed below.

## Current runtime boundary

- The application is native `eframe`/`egui` only. Web/WASM demo, binary, component-model plugin path, and CI target are removed.
- Clipboard capture is generic `arboard` text-or-image polling. Source application, concealed/private markers, generation/provenance, atomic all-flavor enumeration, and OS-history exclusion are unavailable.
- Automatic paste is disabled until a native adapter confirms the destination immediately before injection. Eligible non-sensitive selections are copy-only; sensitive copy is blocked without proven OS-history exclusion.
- One-time passwords, private keys, recovery codes, and explicit skipped-capture recovery use an at-most-32-item process-only history lane with hard expiry. These clips never enter SQLite or import quarantine, cannot be pinned or session-protected, and vanish when the process exits.
- The live SQLite database is unencrypted. Strict security mode may block capture while encryption or required native privacy proof is unavailable.
- UI preferences persist through the root configuration, reduced motion follows the OS while unset, unknown configuration keys fail validation, and action labels expose Paste only when the active delivery backend proves it.
- Migration safety copies are temporary and removed only after the upgraded or next-start live store opens fully and passes `quick_check`; a failed open preserves them. They are not durable rollback backups, and backup-evidence records do not create a backup.
- The native executable plugin protocol is a bounded contract only. No plugin host, OS sandbox, installation path, or clipboard grant is active; execution remains release-gated.

## Status vocabulary

| Status | Meaning |
|---|---|
| **Runtime** | Exercised by the current resident binary, popup, store, CLI, or repository automation. |
| **Foundation** | Implemented and tested as a bounded reusable contract, but not connected to the final user-facing surface. |
| **Adapted** | A narrower implementation preserves privacy, portability, or truthful capability reporting. |
| **Native required** | Completion depends on real per-OS APIs, clients, transports, credentials, or assistive-technology evidence. |
| **Rejected** | The proposed mechanism conflicts with a safety or correctness constraint; its replacement is recorded. |

## Three review passes (iterations)

- [x] **Iteration 1: correctness and privacy.** More than ten independent worker/critic reviews inspected capture, persistence, deletion, import, config, delivery, and plugin boundaries. Corrections included pre-write rejection for unsupported sensitive clipboard history exclusion, the bounded volatile secret lane with hard expiry and volatile undo, immediate snapshot pruning even while the store lock is busy, store/import rejection for memory-only classes, active/archived/expired-safe quarantine restore, fail-closed malformed search, Compose sensitivity reclassification through the capture detector, copy-only generic delivery, and strict unknown-key validation for normal and migration paths. `cargo test --workspace --all-targets --all-features --locked --offline` and the no-default-feature matrix passed.
- [x] **Iteration 2: product design and platform truthfulness.** The 24-theme/DPI/surface golden matrix was regenerated and reviewed through headless WGPU, then passed without update mode. GUI tests cover WCAG AA semantic colors, sensitive masking, query completion, AccessKit list/list-item/combobox/listbox roles, dynamic Copy/Paste labels, and OS-derived reduced motion. Native OS/assistive-technology conformance remains a separate gate.
- [x] **Iteration 3: architecture and release evidence.** The GUI was split into navigation, projection, media, Trust-view, state, design, experience, view, and shell slices; History owns persistent plus volatile lanes; the native executable protocol remains disconnected and gated behind bounded length-prefixed framing. The final critic's nine findings and four-item re-review were resolved, including explicit reduced-motion override intent, config-first compensated autostart, and migration cleanup only after a fully opened live store passes `quick_check`. Strict clippy, full test matrices, release budgets, `cargo deny`, `cargo audit`, `cargo vet`, fuzz-target compilation, workflow YAML parsing, documentation contracts, and whitespace review passed locally.

No iteration is complete merely because code or a contract exists. A checkbox may be marked only after the review ran and its findings were either corrected or explicitly accepted with evidence.
