# Public limitation ledger

Ledger version 2. Reviewed on 2026-07-22 against the current development line. This page describes the executable as it exists, not the intended product. A limitation leaves this table only when its linked gate has reproducible evidence; a unit-test foundation is not enough to claim native behavior.

| ID | Severity | Current limitation | Practical workaround | Exit evidence |
|---|---|---|---|---|
| LIM-001 | Critical | The history database is bundled SQLite and is **not encrypted at rest**; whole-database encryption remains unproven. Owner-only permissions, secure deletion, WAL truncation, sensitive TTL, and the process-only lane for OTP/private-key/recovery-code classes reduce exposure but do not replace SQLCipher. Other eligible clips still persist in plaintext SQLite. | Do not use this build as a secret vault; keep secret detection enabled, use short retention, and wipe history before handing over the machine. | SQLCipher opens only with an OS-keystore key; wrong-key and restart tests pass; raw DB/WAL/SHM/CAS canary scans are clean on macOS, Windows, and Linux. |
| LIM-002 | High | The active `arboard` backend captures plain text and raw RGBA images, not every native clipboard flavor or delayed-render representation. | Keep the source application open and verify rich/RTF/file/custom-format content before relying on the stored copy. | Native format-fidelity corpus and real-app round trips pass on every supported backend. |
| LIM-003 | Critical | Native concealed/transient and password-field hints are not comprehensively available through the fallback backend. Pattern-detected OTP/private-key/recovery-code classes use process-only storage, but missing native metadata can still prevent correct classification before persistence. | Exclude known secret-handling apps, keep strict mode enabled where supported, and avoid copying secrets into an early build. | Per-OS conformance proves every supported native hint is evaluated before persistence. |
| LIM-004 | High | Source-app attribution is unavailable or best-effort on some sessions, especially GNOME Wayland. Unknown provenance can therefore be rejected in strict mode or shown without an app name. | Use content rules and manual capture where reliable source identity matters. | Native owner/focus attribution tests pass for each declared session class. |
| LIM-005 | High | GNOME Wayland exposes no sanctioned background clipboard-monitor protocol; background capture can miss copies. | Use capture-on-summon/manual capture, an X11 session, or a compositor with supported data-control capability. | A reviewed native integration supplies events without fabricated capability or focus theft. |
| LIM-006 | High | Automatic paste-back is disabled. The generic backend cannot confirm the intended destination immediately before injection, so Accessibility/input-injection permission alone is insufficient and all eligible non-sensitive selections remain copy-only. | Select a clip, then paste it manually in the intended application. Sensitive copy is also blocked unless clipboard-history exclusion is proven. | Clean-machine tests prove target confirmation, focus restore, and paste on each OS without wrong-target events. |
| LIM-007 | Medium | Remote-session startup pause is wired, but continuous idle and screen-lock auto-pause still need native event adapters. | Use the tray/menu pause action before lock, screen sharing, or unattended periods. | Live lock/idle/remote transitions and resume policy pass native tests on all supported OSes. |
| LIM-008 | Critical | `vbuff-sync` contains tested protocol, crypto, and device-policy foundations, but there is no live discovery, authenticated transport, durable replication, or two-device UI. | Treat history as local-only and use deliberate OS/file transfer outside vbuff. | Two real devices pair, replicate, revoke, recover offline, and pass the documented threat model. |
| LIM-009 | High | Browser, editor, launcher, automation, MCP, terminal, and webhook code is a bounded contract layer only; no extension or daemon endpoint is enabled. | Use the resident popup and local CLI features that are explicitly documented as active. | Signed clients connect through authenticated daemon dispatch and pass integration conformance tests. |
| LIM-010 | High | The native executable plugin protocol, manifests, recipes, adapters, and typed pipelines are contracts only. The resident app does not install or launch third-party executables, and no OS-sandboxed process host or clipboard grant path is active. | Use only built-in transforms; do not treat third-party plugin bundles as installable. | A per-OS sandboxed, resource-limited host enforces exact grants, signed installation receipts, protocol limits, and kill-on-violation behavior. |
| LIM-011 | High | Durable encrypted delete recovery is not active because no OS-keystore provider supplies the recovery key. | Use the five-second in-memory Undo; after process exit, deletion is final. | Key lifecycle, restart restore, lock, tamper, and residue tests pass per OS. |
| LIM-012 | Medium | Packaging is incomplete: macOS notarization is credential-gated, Windows signing/installers and Linux package formats are not all shipped, and there is no live auto-update install loop. | Build from source or use only artifacts whose checksums and provenance you independently verify. | Clean-install, signing, Gatekeeper/SmartScreen, package-manager, downgrade, and rollback gates pass. |
| LIM-013 | Medium | AccessKit semantics, keyboard reachability, OS-derived reduced motion, WCAG AA semantic colors, and 24 cross-platform headless WGPU goldens are automated, but real NVDA, VoiceOver, Orca, forced-color, RTL, CJK, and native compositor evidence is incomplete. | Prefer keyboard navigation and report the exact OS/assistive-technology combination when filing an accessibility issue. | The manual assistive-technology and native display matrix is attached to a release evidence bundle. |

## Maintenance rule

- Update `Ledger version` whenever an entry is added, removed, split, or materially reworded.
- Keep removed IDs reserved and record their closing release in git history; never renumber active limitations.
- A workaround must be usable today. A planned feature is not a workaround.
- Release notes link this ledger and the evidence bundle. Any `Unknown` release gate remains a limitation, never an implied pass.
