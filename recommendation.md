## Current implementation snapshot

- The shipping surface in this repository is native `eframe`/`egui` only. Web/WASM demo and build paths are removed.
- Clipboard capture is generic `arboard` polling for text or image. Source identity, concealed/private markers, clipboard generation/provenance, atomic flavor enumeration, and OS-history exclusion are unavailable rather than inferred.
- Automatic paste is disabled until a native adapter confirms the destination immediately before injection. Eligible non-sensitive selections are copy-only; sensitive copy is blocked without proven OS-history exclusion.
- One-time passwords, private keys, recovery codes, and explicit skipped-capture recovery live only in a bounded, hard-expiring process-memory lane. They never enter SQLite/import, cannot be pinned or session-protected, and disappear on exit.
- The live SQLite database is unencrypted. `strict_security_mode` may block capture while encryption or required native privacy capabilities are unavailable.
- UI preferences persist through the root configuration; reduced motion follows the OS when unset. Unknown configuration keys fail closed instead of silently accepting misspelled security or UI settings.
- Migration safety copies are temporary and removed only after the upgraded or next-start live store opens fully and passes `quick_check`; a failed open preserves them. Backup-evidence APIs are not a durable rollback or backup service.
- The native plugin executable protocol has bounded length-prefixed JSON framing but is contract-only. There is no active process host, OS sandbox, installation path, or clipboard grant; all plugin execution remains release-gated.

## 1. Strategic positioning and the differentiation bets to commit to

### The market changed

Windows and macOS now provide built-in clipboard history. Paste advertises OCR, Teams, shared pinboards, Apple Intelligence, local MCP, and a lifetime option. CrossPaste advertises cross-OS LAN E2EE, OCR, MCP, and a browser extension. PowerToys, CopyQ, Raycast, Maccy, PastePal, and PasteBar already cover most isolated transform, script, queue, collection, and privacy checkboxes.

The old claim that every competitor owns at most two of four corners is both stale and too absolute to verify. Cross-platform reach, polish, encryption, sync, OCR, and MCP remain useful requirements, but they are not a durable position by themselves.

### Initial segment

Build first for developers, DevOps/security engineers, and technical support staff who use at least two desktop operating systems and move code, paths, logs, tables, customer data, and secrets among browsers, IDEs, terminals, remote sessions, ticketing tools, and AI assistants.

### The four systems to defend

1. **Open Format Fidelity Lab.** Publish a versioned source-app to target-app corpus, preserve all offered flavors atomically, and report supported, degraded, and unsupported behavior. Compatibility evidence and app adapters compound over time.
2. **Contextual Recall.** Rank and browse by privacy-safe source application, time, session, content type, and neighboring copies. Text and semantic similarity are signals, not the whole retrieval model.
3. **Clipboard policy and graded delivery evidence.** Apply one typed policy to capture, storage, disclosure, sync, and paste. Distinguish `Staged`, `TargetConfirmed`, `InjectionSent`, and integration-only `ApplicationAcknowledged`; never claim destination-only secrecy after staging bytes on a global clipboard.
4. **Scoped Context Gateway for AI.** Expose zero clips by default. Grants are client-, collection-, class-, operation-, byte-, and time-scoped, with secret denial, preview, revocation, provenance, and redacted audit.

The immediate order is: connect egui to paged full-history retrieval, prove one narrow Windows 11 native vertical, then publish the app-pair fidelity matrix and test contextual ranking. SQLCipher, native privacy markers, keyboard/accessibility correctness, bootstrap recovery, import/export, and reliable autostart are release foundations rather than differentiators. Explicit E2EE directed handoff must precede ambient full-history sync. Generic OCR, LLM rewrite palettes, shared boards, mobile peers, MCP, and a plugin marketplace stay frozen until the native beta and demand gates pass.

### Positioning non-goals

- **Not a feature-count race.** Do not implement a checkbox merely because a competitor has it.
- **Not a cloud product by default.** No vendor backend holds clipboard data by default; a future relay remains optional and blind to payload plaintext.
- **Not an unscoped AI memory server.** A local transport does not make broad clipboard access safe.
- **Not a guarantee against a privileged local attacker.** Root/admin, an attached debugger, and kernel compromise remain outside the threat model.
- **Not a clipboard editor at capture time.** Stored canonical bytes remain immutable; previews and transformations are derived at action time.

## 2. Build now / build later / skip matrix

"Build now" means the full-history slice and admitted single-process Windows vertical (`vbuff-types`, `vbuff-core`, `vbuff-store`, `vbuff-platform`, `vbuff-gui`, plus the root binary). `vbuff-daemon`/`vbuff-ipc`, `vbuff-sync`, and `vbuff-cli` activate only after their post-beta gates. "Build later" means demand-gated; "skip" means explicitly out of scope or a trap.

| Feature area | Verdict | Tier | Justification |
|---|---|---|---|
| **Capture: Windows native event path** (`WM_CLIPBOARDUPDATE`, sequence/owner/format evidence) | **Build now** | Windows alpha | This is the first proof vertical. `arboard` is a degraded text-or-image fallback, not a source for native fidelity or privacy claims. Other OS adapters enter only through M11 demand and the same evidence battery. |
| **Capture: ordered items and all offered flavors byte-for-byte** | **Build now** | Windows alpha | `ClipboardSnapshot -> ClipboardItem[] -> Flavor[]`, native IDs, canonical bytes, BLAKE3 dedup, and visible downgrade are core fidelity. A flattened list cannot support the headline claim. |
| **Capture-decision gate (fail-closed)** | **Build now** | MVP | Concealed/transient hints, app exclusion, incognito, size cap, whitespace skip - every privacy promise routes through one ordered, testable gate before disk. |
| **Store: SQLite + SQLCipher, WAL, txn-per-capture, count cap, pins exempt** | **Build now** | MVP | WAL, transactions, migrations, dedup, FTS5, and CAS are present; SQLCipher plus OS-keystore keying remains the non-negotiable release gap. |
| **Store: out-of-row blob CAS, total-size cap, thumbnails, image transcode** | **In progress** | MVP/v1 | Refcounted sharded CAS is now present. Total-byte retention, canonical thumbnails, and transcode remain v1 hardening. |
| **Search: substring as-you-type + live highlight + type/pinned filters** | **Build now** | MVP | Keyboard-first recall is the core loop. egui `ScrollArea::show_rows` gives row virtualization for free. |
| **Search: FTS5 indexed, fuzzy, regex, relevance ranking** | **Build later** | v1 | Needed at 30k-50k items (CopyQ/Maccy collapse here), but the SQL infra is staged in MVP; switch the tier on in v1. |
| **Paste-back into prior focus** (Windows `SendInput` first) | **Build now** | Windows alpha | Capture focus first, reconfirm immediately before injection, and fall back to copy-only for changed/elevated/unproven targets. Other native injectors require their own promoted adapter. |
| **Paste: plain-vs-rich, paste-stack, terminal-safe combos, restore-after-paste** | **In progress** | MVP/v1 | The current ephemeral Compose stack and merge templates are live, with an explicit safe form-step foundation. Destination-aware native formatting, FIFO persistence, focus verification, and restore-after-paste remain v1. |
| **Snippets + text expansion** (abbreviation, folders, date/time placeholders) | **Build later** | v1 | The feature catalog tags a minimal set MVP, but expansion needs reliable secure-input handling and keystroke timing first. **Recommend deferring the abbreviation-expansion path out of the first shippable build** to keep the MVP focused on the copy->store->paste loop; saved snippets (pinned permanent clips) ship in MVP, expansion follows in v1. |
| **Transformations / quick actions** (case, trim, strip formatting, regex) | **Build later** | v1 | One-shot "paste as plain" is cheap and should be MVP-adjacent; the broader transform palette is v1. Never apply transforms to stored canonical bytes. |
| **Organization: pin, favorite, permanent, type filter** | **Build now** | MVP | Pinning is the most-wanted persistent-bank feature (Maccy, Win+V, Raycast). Pins must be a separate eviction-exempt class. |
| **Organization: tags, collections, named tabs, notes, drag-reorder, color-code** | **Build later** | v1/v2 | Depth that distinguishes a daily driver; not needed for first use. Pinboards/collections are a known Paste strength worth matching in v1. |
| **Tray / menu-bar item + autostart + single-instance guard** | **Build now** | MVP | The watcher must be live within a few hundred ms of login and survive popup dismissal. Even in the single-process MVP, the resident process + single-instance guard is required. |
| **Daemon/IPC formal split** (`vbuff-daemon`, `vbuff-ipc`) | **Build later** | v1 | The MVP runs watcher + store + GUI in one root-binary process. Promote to the formal daemon/IPC split when the CLI and external pickers need to reach the live watcher. |
| **CLI** (`vbuff-cli`: list/get/copy/add/search), external pickers, `vbuff://` | **Build later** | v1 | Scriptability is a core differentiator vs polished-but-closed rivals, but it depends on the IPC layer; sequence after the single-machine core is stable. |
| **Sync** (`vbuff-sync`: authenticated pairing and explicit directed handoff first) | **Freeze; experiment only after demand** | Post-beta | CRDT/HLC, envelopes, membership, policy, anti-entropy, recovery, receipts, capabilities, and padding are isolated foundations, not a product. If M6 demand justifies transfer, ship one explicitly selected, TTL-bound E2EE text handoff before considering ambient history replication. |
| **AI features** (semantic search, OCR, summarize/rewrite/translate, MCP server) | **Scoped gateway only after prerequisites** | v1-future | Capture emits fail-closed eligibility and bounded contracts exist, but no model host, OCR, MCP server, or external endpoint is active. Generic AI actions are commodity. The only strategic path is a least-privilege context gateway after encrypted storage, native privacy proof, and live authorized IPC; it exposes zero clips by default. |
| **Collaboration / team** (shared snippet libraries, shareable links, roles, expiry) | **Build later** | v2 | The TextExpander niche. Lands last, after sync and conflict resolution are solid. |
| **Mobile (iOS/Android) as synced peers** | **Build later** | post-v2 | Useful reach, but no longer a unique differentiator. It depends on proven directed handoff, sync policy, and platform share-sheet workarounds. |

### Four opinionated calls worth stating explicitly

1. **Pull abbreviation-based text expansion out of the first shippable MVP.** Saved/pinned permanent clips ship in MVP; live keystroke expansion (the Espanso job) is harder to get right than it looks (secure-input interaction, timing, word-boundary/case propagation) and risks the polish bar on the core loop. Ship it in v1 when the paste/keystroke infrastructure is hardened.
2. **The single-process MVP still needs a resident, supervised capture component.** Even before the formal `vbuff-daemon`/`vbuff-ipc` split, the root binary must hold the clipboard watcher, the single SQLite writer, the global hotkey, and the tray on background threads while the egui GUI runs on the main thread, guarded by a single-instance lock. "Capture only while a window is open" is a top competitor failure (CopyQ on Wayland/XWayland).
3. **Keep LAN/internet sync out of the active roadmap and say so loudly.** Stabilize the private single-machine core first. After beta demand, sequence explicit authenticated handoff of selected clips before ambient history replication so retention, wrong-device, expiry, and revocation policy can be proven on a smaller surface.
4. **Skip the in-MVP launcher/macro engine and the user-scripting DSL.** Do not become CopyQ. Power is delivered as discoverable GUI actions plus a later CLI, not a JavaScript/C# scripting console on the hot path.

### Design direction and product cut line

vbuff's design should feel more like a precise resident instrument than a content app: quiet, dense, fast, trustworthy, and immediately usable. The strongest design move is restraint. The popup is the product; it should open, filter, expose state, and paste without a tour or a decorative shell.

| Surface | Design rule | Cut line |
|---|---|---|
| Popup | Stable row heights, strong selected state, small badges, zero layout jump while typing | Cut decorative animation before cutting keyboard speed |
| Row actions | Icons plus tooltips for common tools; explicit text for destructive actions | Cut rare row actions into a command palette before crowding every row |
| Sensitive content | Mask by default, show reason, allow deliberate peek, never use panic styling | Cut visual cleverness before cutting trust clarity |
| Settings | Dense, grouped controls with search and keyboard navigation | Cut deep customization before cutting privacy defaults |
| Tray/menu bar | Recognizable clipboard icon; same command names as popup/CLI: Show, Copy latest, Clear history, Pause/Resume, Start at login, Quit | Cut extra menu items before duplicating command semantics |
| Onboarding | One successful copy -> popup -> paste-back loop | Cut explanatory tours before cutting the first success path |

Design review checklist:

- Can the user identify the selected clip within 100 ms?
- Does every privacy state have a visible, plain-language reason?
- Does text fit in compact mode and high-contrast mode?
- Can every critical workflow run without a mouse?
- Are destructive actions unmistakable and undoable where possible?
- Does the UI distinguish current implementation from target/future capability?

The SOLID/DRY product rule: a command should have one name, one behavior, and one permission model across popup, tray, CLI, IPC, and docs. If "clear history" means different things in two surfaces, the design is wrong even if the code compiles.

Current implementation review: the popup uses one design-token module for type, spacing, control sizes, radii, theme-aware semantic colors, and fixed-size icon actions with tooltips. Navigation, projection, media, and Privacy rendering are separated from the main egui shell. Popup and tray keep one typed capture-health vocabulary. History and Stack are the primary destinations; Privacy and Settings remain secondary. History provides local search, completion, actionable misses, expiry, masked sensitive rows, capacity-cleanup exceptions, and a bounded process-only lane for secret classes that must never reach SQLite. Single click selects, double click executes, Enter executes the selected clip, and Tab/Right Arrow accepts completion. Source-dependent UI remains unavailable because the generic backend cannot prove source identity. Stack is an ephemeral focused tool without automatic focus injection. Twenty-eight checked-in headless WGPU goldens cover normal, minimum, and wide layouts across themes and `1x`/`1.5x`/`2x` DPI; they are visual-regression fixtures, not native OS or assistive-technology evidence. Theme-aware semantic and secondary colors have WCAG AA checks, strong boundaries have non-text contrast checks, AccessKit roles model list and completion semantics, UI preferences persist, and reduced motion follows the OS until the user overrides it. The privacy posture remains in `Needs attention` without both encrypted storage and sensitive-memory-only handling; current-session evidence explicitly avoids security-guarantee wording. Tray/menu-bar glyphs distinguish active, paused, degraded, locked, and future `PasteSent` states; the current generic runtime never emits delivery success because automatic paste is disabled. The pause-aware heartbeat/watchdog reports stalled capture without treating an intentional pause as a failure. Schema 7 archive, retention, manifest, quarantine, restore, residency, backup-evidence, export, and legal-hold APIs remain foundations until focused user workflows own them. Native capture/privacy proof, target confirmation, history exclusion, continuous idle/lock signals, screen-reader verification, SQLCipher, real compositor evidence, and a sandboxed plugin host remain release work. The [native egui design review](docs/design-review-native-egui.md) records the ten-role panel and three implementation iterations.

## 3. Top competitor mistakes to avoid, by theme

Each row names the competitor(s) and maps to a concrete vbuff decision. Grounded in `docs/mistakes-top-500.md`, `docs/pain-points.md`, and the architecture's own "How vbuff avoids competitors' mistakes" table.

### Theme A - Silent data loss (the worst category)

| # | Mistake (who) | vbuff decision |
|---|---|---|
| 1 | **Clip lost when the source app/window closes** - the single most-reported Linux frustration (X11/Wayland core, GPaste, Klipper, Diodon, Clipman, CopyQ) | Eagerly materialize all offered MIME types into vbuff's own store the instant a selection event fires; on X11 take CLIPBOARD ownership and re-serve. vbuff *is* the persistence helper (built-in wl-clip-persist behavior). |
| 2 | **Capture only runs while the UI window is open / dies on XWayland window close** (CopyQ, cliphist, GPaste) | A supervised, always-on resident capture path decoupled from any window; a hidden persistent selection/message window keeps X11 capture alive. |
| 3 | **Missing rapid successive copies** | Capture synchronously inside event-driven handlers and persist on a worker. On polling APIs, compare sequence counters and expose gaps that cannot be reconstructed. The CI metric is zero silent loss, not an impossible universal recovery claim. |
| 4 | **History wiped or pins lost on app update** (Ditto silently emptied history on 3.21.134; Maccy lost entire history incl. pins) | The on-disk store is a version-independent contract: forward-only transactional migrations gated by `PRAGMA user_version`, automatic pre-migration backup, refuse to start rather than wipe on failed migration. |
| 5 | **Capture silently dies** - listener crash, broken viewer chain, GNOME Wayland unsupported, second manager fighting for the chain (Win+V service crashes, ClipboardFusion, CopyQ, Ditto) | Supervised capture with a heartbeat/watchdog that re-registers OS hooks (modern `AddClipboardFormatListener`, never the fragile `SetClipboardViewer` chain), plus an always-visible "capturing / paused / unsupported" health indicator so silent loss is impossible. |

### Theme B - Privacy and secret leakage

| # | Mistake (who) | vbuff decision |
|---|---|---|
| 6 | **Ignoring concealed/transient hints, storing passwords/2FA** | Check native hints before persistence. On Windows, model `ExcludeClipboardContentFromMonitorProcessing`, `CanIncludeInClipboardHistory`, and `CanUploadToCloudClipboard` separately so monitor, local-history, and cloud-upload policy are not collapsed into one guess. Zero forbidden bytes reach disk, sync, temp files, or logs. |
| 7 | **No default exclude list, so password managers leak** (Ditto, Flycut) | Ship a sane default deny-list (1Password, KeePassXC, Bitwarden) plus per-app include/exclude, applied before persistence. Defense in depth: hints + secure-field detection + app exclusion + secret detectors + regex + incognito. |
| 8 | **Plaintext, world-readable history; recoverable "deleted" secrets in freelist/WAL** (Ditto banned by a security team; CopyQ default; Win+V) | SQLCipher full-DB encryption, key in the OS key store (never beside the DB); ACL-restricted files; `secure_delete` + WAL-truncate + `VACUUM` to scrub; whole-artifact canary scans on Windows and independently on every later promoted OS. |
| 9 | **OCR-on-everything indexes secrets** (Clipory makes a copied receipt fully searchable) | OCR/AI features are opt-in and off by default; sensitive items get masking, shorter retention, and sync exclusion. We do not index image text without explicit consent. |
| 10 | **Telemetry from a tool over your most sensitive data** (ClipboardFusion opt-out telemetry; CopyClip ads + phone-home) | Zero telemetry by default; open-source core so the claim is auditable; any future analytics is strictly opt-in. |

### Theme C - The paste action itself failing

| # | Mistake (who) | vbuff decision |
|---|---|---|
| 11 | **Auto-paste silently fails / inserts nothing / shows a notification of the text instead of pasting** (Maccy #1230/#1280, Ditto #245, ArsClip, ClipMenu) | Restore focus and confirm the target is frontmost before injecting; surface a clear message on detected failure instead of a silent no-op or a notification dump. |
| 12 | **Wrong character pasted / held modifiers leak** (Ditto injects a literal "v", #649) | Clear physical modifier state before injecting; use OS APIs (`CGEvent`/`SendInput`/`XTEST`) directly for ordering control rather than relying on a generic injector. |
| 13 | **Paste lands in the wrong window** (CopyQ on Wayland #2136, cliphist) | Capture the focus token first; on Wayland where injection is unavailable, set-and-let-user-paste rather than firing into the wrong target. Silent wrong-target is worse than a no-op. |
| 14 | **Popup steals focus and breaks the paste target** (Maccy #144, ClipMenu, Diodon) | Capture focus before showing the popup; the popup is a transient that restores focus on dismiss. |
| 15 | **Wrong formatting on paste; "paste without formatting" mangles content** (Maccy #109/#693, Alfred, Paste) | Store every co-offered representation atomically; decide plain-vs-rich at paste time; prefer `image/png` over an `<img>` HTML tag. |

### Theme D - Performance and resource collapse at scale

| # | Mistake (who) | vbuff decision |
|---|---|---|
| 16 | **Whole history rendered on every open; freezes at 30k-50k items** (Maccy #1097, CopyQ) | Virtualized, keyset-paged list (egui `show_rows`); never `SELECT *` the table. Sub-frame search-as-you-type at 100k+ items is a `criterion` benchmark gate. |
| 17 | **Unbounded SQLite growth; deleted rows never reclaimed -> multi-GB DBs** (Ditto reports of 14GB-216GB) | Hard count + total-byte ceilings with continuous eviction; `auto_vacuum=INCREMENTAL` + periodic `VACUUM`; current DB size shown in settings. |
| 18 | **Raw uncompressed Retina TIFFs (10-30 MB) bloat the DB and pin a core** (Ditto, Maccy, GPaste #40 loops re-adding a screenshot) | Transcode large images to compressed PNG; store oversized blobs externally by content hash with only a thumbnail in the row; refcounted GC. |
| 19 | **Memory leak / idle CPU spin / polling instead of events** (Maccy linear leak #384, ~45% idle CPU #993; CopyQ filesystem polling #1131) | Event-driven capture everywhere the OS allows; poll only the `changeCount` integer on macOS with adaptive idle backoff; near-0% idle CPU as a multi-day metric. |
| 20 | **Capture path blocks/OOMs on huge payloads** (CopyQ #3096: 700MB copy -> bad_alloc, monitor crash) | Stream bytes off the event thread with a configurable size cap; above the cap store a preview + external reference; survive allocation failure without killing capture. |

### Theme E - Linux/Wayland and platform-specific traps

| # | Mistake (who) | vbuff decision |
|---|---|---|
| 21 | **Relying on a protocol GNOME Mutter refuses to implement -> empty history on GNOME Wayland** (cliphist, clipman, wl-clipboard) | Probe compositor capability at startup; degrade to capture-on-summon + manual hotkey on GNOME Wayland; surface a per-compositor capability badge. Never fake capability (a stated non-goal). |
| 22 | **PRIMARY vs CLIPBOARD confusion floods or misses copies** (X11 core, GPaste, Klipper, CopyQ) | Treat PRIMARY and CLIPBOARD as distinct; capture CLIPBOARD by default; PRIMARY is opt-in and hard-debounced so mere selection does not flood history. |
| 23 | **Echo loops re-capturing our own restored/written clips** (GPaste, Clipy, CopyQ; one image pins a core) | Fingerprint (BLAKE3 + owner) every self-write with a short suppression window + sentinel format; on a self-write match, bump the existing row's timestamp at most, never insert. |
| 24 | **Routing the live DB through a cloud-sync folder -> `.conflict` files and corruption** (Ditto on Drive/Dropbox) | DB in a per-machine owner-only app-data path; sync at the record level through a real conflict-resolving protocol, never the raw SQLite file; warn at startup if the DB path is inside a known cloud folder. |
| 25 | **macOS Accessibility permission silently drops after updates and paste dies** (Maccy #159/#835, Flycut, Clipy); **login-item silently unchecks** (Flycut #206) | Detect `AXIsProcessTrusted()` false and fall back to copy-only with a banner that deep-links to the pane; verify autostart registration on launch; never assume a once-granted permission persists. |

Cross-cutting guarantee behind the whole table: the pure behavioral suite runs against fakes everywhere, while each promoted native adapter must produce its own real-host evidence. The active release lane is Windows; macOS, X11, and Wayland are not inferred from shared traits. `vbuff-core` holds dedup/fingerprint/retention/concealment policy as testable logic, and the resident process exposes visible capture health so silent observed-state loss cannot hide behind a green unit suite.

### Band 3 - Low / speculative impact, pursue only opportunistically or as future

These are real but niche; do not let them pull focus from the wedge.

- Application-aware auto-switching clip groups (#52), group-copy bundles (#55), named clipboards (#74) - power-user niche, v2 at most.
- AI summarize/rewrite/translate/captioning (#3, #4, #5, #11), on-copy math (#12), code explanation (#14) - v1-future, opt-in, on-device-first.
- Clipboard-as-MCP-memory-layer (#15, #111) - a genuinely novel future bet, but only after the core is a daily driver.

### Explicit traps to skip (or hard-gate)

| Trap (extras #) | Why skip / gate |
|---|---|
| **OCR-on-every-copied-image** (#2, #42, #101, #109) | Turns copied receipts/IDs/cards into searchable plaintext secrets (the Clipory failure). If built at all: opt-in, off by default, on-device only, excluded from sync. |
| **User-scriptable JS/C# transform engine with auto-triggers** (#37, #43, #108) | This is the CopyQ trap - power gated behind a DSL that casual users bounce off, and an arbitrary-code execution surface on the clipboard hot path. Deliver power as GUI actions + a typed CLI instead. |
| **Cloud sync via the user's own Google Drive / cloud-drive file** (#61) | The 1Clipboard/Ditto failure: `.conflict` files, corruption, and a dead backend. Record-level E2E sync, never raw-file-in-a-synced-folder. |
| **Web fallback / no-install pickup link** (#86), **background-capture entitlement abuse on iOS** (#94) | Either widens the trust surface (a relay that can see content) or fights the OS sandbox. Off the roadmap. |
| **Structured-data extraction + auto field-mapping on paste** (#18), **web-scraper clips** (#51), **list-to-checklist** (#13) | Scope creep into RPA/automation. Out of scope for a clipboard manager; revisit only if a clear user pull appears. |
| **Latest-clip injection into the mobile keyboard** (#91) | Platform-fragile, keyboard-extension-specific; future at best. |

## 5. Key risks and mitigations

Building on the architecture's risks table, with two feasibility spikes called out explicitly.

| Risk | Likelihood / Impact | Mitigation |
|---|---|---|
| **egui native-feel is below the polish bar** the wedge demands. egui draws its own widgets and can look detached from the host OS. | Medium / High | Judge the real Windows popup, not a component gallery: full-history list, compact states, keyboard flow, DPI/text scaling, high contrast, and NVDA. Keep one token system and stable geometry; use a shaped-content layer where egui text is insufficient. A toolkit rewrite needs measured workflow failure, not aesthetic preference alone. |
| **Network work consumes the product before the native wedge is proven.** NAT traversal, relay metadata, membership, revocation, and convergence can absorb the roadmap while competitors still beat basic recall/fidelity. | High / High | Freeze transport and replication through M6. If demand survives, M9 proves one explicit LAN handoff with TTL, replay protection, crash-safe one-shot state, and receipt. Ambient sync gets a separate later decision and threat model. |
| **A Wayland compositor advertises no supported data-control protocol; important Mutter configurations remain degraded.** | High / High | Probe `ext-data-control-v1` first and deprecated `wlr-data-control` only as a legacy fallback. Otherwise degrade to capture-on-summon + manual hotkey with honest per-compositor capability state. |
| **Wayland hides foreground-app identity** -> per-app exclusion and private-browser skip cannot function. | High / Medium | Make fail-closed-where-required configurable; show capability badges so users are not lulled into false safety; content/regex/secret rules still work and are surfaced as the active protection. |
| **Encryption silently not engaged** (wrong PRAGMA, fallback to plain SQLite, blob written before sealing). | Low / Critical | Run whole-artifact canary scans on the active Windows release lane and repeat them independently for every promoted adapter; pin cipher params in a config record. Any hit is a release blocker. |
| **Sensitive content leaks** because a source app sets no concealed hint. | Medium / High | Defense in depth: hints + secure-field detection + app exclusion + secret detectors + regex + incognito; sensitive items masked, shorter retention, sync-excluded. |
| **Crate-maturity assumptions wrong** (`tray-icon` SNI, `global-hotkey` Wayland gap, `keyring` v3, SQLCipher PRAGMA defaults, objc2/windows-rs signatures). **Confirmed, not hypothetical:** `cargo audit` against the actual `Cargo.lock` today reports 10 live RUSTSEC advisories - 7 "unmaintained" for the GTK3 chain (`gtk`/`gdk`/`atk` 0.18.2 and their `-sys`/`-macros` crates) pulled in transitively by `tray-icon`'s default-on Linux backend, plus `proc-macro-error` and `ttf-parser` unmaintained, plus an actual **unsoundness** advisory (RUSTSEC-2024-0429) in `glib` 0.18.5's iterator impls, also via the GTK3 chain. See `docs/problems-improvements-top-500.md` #71-73. | Medium / Medium (now: confirmed present) | Verify every platform crate against pinned versions in the Phase-0 spike before locking the stack; the trait boundaries make swapping an impl cheap. Add `cargo audit` to the pre-PR checklist and CI now (`docs/problems-improvements-top-500.md` #75) rather than deferring dependency-risk discovery to a future spike - the GTK3 chain in particular is worth evaluating for a lighter Linux tray backend before v1. |
| **Self-write feedback loop** re-captures our own paste or a restored sensitive clip. | High if unhandled / Medium | Sentinel format + own-ownership tracking + short-TTL content-hash flag; debounce swallows the echo; gate #0 ignores tagged writes. |
| **DB corruption / power loss.** | Low / High | WAL + `synchronous=NORMAL`, txn-per-capture, `integrity_check` at startup, quarantine-and-restart on unrecoverable corruption, online-backup before migrations. |
| **Scope sprawl** across a 640-feature catalog + 122 extras. | High / Medium | Treat the catalog as evidence, not scope. Execute full-history recall -> Windows proof -> fidelity/context beta; every later capability needs an owner, demand signal, smallest slice, and kill metric. |
| **Single-machine MVP regresses on the daemon promise** - capture stops when the GUI closes, or two instances fight for the hotkey/DB. | Medium / High | Even in one process, run watcher/store/hotkey/tray on background threads with a single-instance guard; capture must outlive the popup. This is the bridge to the later formal `vbuff-daemon`/`vbuff-ipc` split. |
| **Trust/abandonment perception** for a new entrant in a graveyard-heavy market (ClipX, 1Clipboard, Clipt, ClipMenu). | Medium / Medium | Open-source core, public roadmap, zero telemetry, no subscription bait-and-switch, visible maintenance cadence. |

## 6. Where the shipped code has already diverged from this document

This section exists because section 2 makes explicit "non-negotiable" calls that the current single-process MVP
binary does not follow, and because some items below were stated as settled fact when they are not yet true in the
repository. Full evidence (file + line) lives in [docs/code-audit-top-50.md](docs/code-audit-top-50.md); this is the
short version so the divergence isn't silently lost between documents.

- **Section 2 calls `arboard`-based polling "the wrong tool" and native event-driven capture "non-negotiable."** The shipped `main.rs` polls `arboard` on a fixed timer on every OS, exactly the pattern this document rejects (audit #11).
- **Step 2 of "the 10 things" (below) states "FTS5 schema present (substring is the shipped tier)" as fact.** No FTS5 table exists in the schema; only three plain B-tree indexes do (audit #21).
- **Step 1 of "the 10 things" calls for `MockBackend` implementations of the four platform traits.** None exist; `vbuff-platform` ships only real OS-backed implementations, so the OS-facing code has no automated tests (audit #34).
- **Step 8 ("Privacy floor") and Theme B above assume encryption, concealed-hint honoring, and a working default deny-list land alongside the MVP.** None of the three exist yet, and the app-exclusion feature that does exist never triggers because the capture backend never reports the source app (audit #1-6).
- **This list should shrink, not grow.** As each gap actually closes in the code, strike the corresponding line here and in `docs/code-audit-top-50.md` rather than letting both silently drift from reality again. First entries struck: audit #22/#23/#28 (the dead `Store::search` path was deleted rather than left to rot) and #24 (the eviction-policy duplication was not merged, but a parity test now guards it - and immediately caught a real bug: `vbuff_core::eviction::evict` was silently sorting by `ClipId` instead of last-touched time, so a re-copied clip could be evicted ahead of a genuinely stale one; fixed by adding `ClipMeta::updated_at`, which the type never carried before).
- **The codebase was also split by SOLID/DRY** (see `architecture.md`'s "Current module map" section and `docs/problems-improvements-top-500.md`): every crate's largest files (`vbuff-store/src/lib.rs`, `vbuff-gui/src/app.rs`, `src/main.rs`) were broken into single-responsibility modules, and the one confirmed duplicate implementation found in the codebase (`parse_rgba_dims`, copy-pasted between `vbuff-platform` and `vbuff-gui`) was consolidated into `vbuff-types`.

## 7. If we only do 10 things for the MVP (dependency-ordered)

Single-process build using `vbuff-types`, `vbuff-core`, `vbuff-store`, `vbuff-platform`, `vbuff-gui`, and the root binary. Each step unblocks the next.

1. **Workspace + types + trait skeleton + mock backends.** Stand up the Cargo workspace, `vbuff-types` (`Clip`/`Flavor`/`ContentKind`/ids), the four `vbuff-platform` traits, and `MockBackend`. Blocks everything; makes `vbuff-core` testable headless on any host.
2. **Encrypted store with txn-per-capture (`vbuff-store`).** SQLite + SQLCipher (full-DB, key in OS keychain), WAL, schema v1 + forward-only migrations, BLAKE3 content hash with a golden-vector test, FTS5 schema present (substring is the shipped tier). Depends on (1).
3. **Core engine: dedup, move-to-top, count-cap eviction, capture-decision gate (`vbuff-core`).** Pure logic over fakes; pins/permanent exempt from eviction; the fail-closed gate (paused/incognito/concealed/secure-input/excluded-app/size/whitespace). Depends on (1)+(2). This is the test crown jewel - proptest byte-fidelity + fail-closed invariants.
4. **One native capture backend, byte-for-byte, all flavors, debounced.** Start with a deliberately narrow Windows 11 scope, subscribe to native events, preserve item/flavor/native-format topology, honor monitor/history markers, and dedup by canonical hash. Depends on (1)-(3). This is the first end-to-end "copy lands in the store with evidence" moment.
5. **Resident process + single-instance guard + tray + autostart.** Watcher/store on background threads, global hotkey live within a few hundred ms of login, tray status item, single-instance lock with handoff. Depends on (4). Guarantees capture outlives the popup.
6. **egui popup with virtualized list + substring search-as-you-type + live highlight.** `ScrollArea::show_rows`, type icons, image thumbnails, empty/no-results states, pinned/favorite badges, full keyboard nav, dark/light, DPI. Depends on (5) and the polish spike from section 5. This is the face of Bet 1.
7. **Paste-back into prior focus.** Capture focus *first*, write flavors, dismiss popup, restore focus, inject the paste combo (CGEvent/SendInput/XTEST), with copy-only fallback where injection is unavailable. Enter / double-click / number-key. Depends on (6). Now it is actually a clipboard manager.
8. **Privacy floor: default app deny-list, incognito/pause, encrypt-at-rest verified, secure-delete.** Default-exclude known password managers, incognito flag, the canary-grep at-rest CI test green on the primary OS, `secure_delete` + WAL-truncate scrub. Depends on (3)+(2). Bet 3 made real.
9. **Organization basics: pin, favorite, promote-to-permanent, type filter, clear-all (pin-protected), delete-item.** Pins as a separate eviction-exempt class with stable order. Depends on (6). The persistent-bank feature every rival's users ask for.
10. **Second native backend after demand evidence.** Add macOS with the same conformance and fidelity runners only after the Windows beta passes activation, retention, payment-intent, loss, residue, and wrong-target gates. X11 and each Wayland compositor remain separate evidence rows; unsupported sessions degrade visibly. Do not claim cross-platform parity from shared traits alone.

Before this dependency-ordered native list, wire the popup to paged DB-backed retrieval so clips older than the current 1,000-row snapshot are reachable. Everything else - fuzzy/regex tiers, image transcode, snippets/text expansion, the transform palette, daemon/IPC, CLI, MCP, plugins, and sync - waits behind the first native beta and its demand gate.

## Ideas and improvements backlog (product & strategy)

> Items 114-197 of a 600-idea backlog. Companion lists: engineering ideas (1-113) in [architecture.md](architecture.md), user-facing/operations ideas (198-300) in [docs/ideas-top-300.md](docs/ideas-top-300.md), extended ideas (301-400) in [docs/ideas-301-400.md](docs/ideas-301-400.md), review backlog items (401-500) in [docs/ideas-401-500.md](docs/ideas-401-500.md), and evidence-backed ideas (501-600) in [docs/ideas-501-600.md](docs/ideas-501-600.md). Effort tags: `S`/`M`/`L`.

### Differentiation & positioning bets

114. **Provable-private mode with a live, auditable privacy ledger** `[M]` - Ship an in-app, tamper-evident log of every capture decision (captured / skipped-because-concealed / skipped-because-excluded-app / masked) plus an at-rest cipher-state readout, so users can SEE that secrets were never written rather than trusting a checkbox. _Value: Every rival asks you to trust an opt-out toggle; none can show you what it actually did. Turning the recommendation doc's fail-closed gate and canary-grep test into a user-facing, verifiable artifact converts vbuff's strongest internal guarantee into a marketable, screenshot-able trust wedge that Ditto (plaintext DB), Win+V, and CopyQ structurally cannot match._
115. **Format-fidelity contract with a round-trip self-test badge** `[M]` - For each clip, store every co-offered MIME flavor and run a continuous background round-trip check (capture -> store -> re-serve) that surfaces a green 'lossless' or amber 'degraded on this platform' badge per item and per app. _Value: The single most-reported cross-platform complaint is silent format loss (Linux text/plain-only, macOS 'paste without formatting mangles content', browser image-as-HTML). No competitor exposes fidelity as a measured, visible guarantee; making 'never silently downgrades your copy' a checkable claim is a defensible moat against CopyQ, cliphist, Maccy, and Alfred all at once._
116. **Publish parity evidence only after a second native adapter exists** `[L]` - Keep one canonical intent/keymap behavior spec now, but do not show a cross-platform parity report until at least two real adapters pass the same app-pair, focus, privacy, accessibility, and packaging gates with native modifier rendering. _Value: shared key bindings are useful architecture, but claiming four-target parity from fake-backed CI would turn an unproven aspiration into misleading product UI._
117. **Backend-portable encrypted history vault (no vendor you can kill)** `[M]` - Define an open, documented, E2E-encrypted on-disk export/restore format that a user can park on ANY transport (a USB stick, their own S3/WebDAV, git, a NAS) and restore byte-identical on another machine, fully decoupled from vbuff's own relay. _Value: The graveyard (Clipt, 1Clipboard) burned users when a backend died; the docs name 'never a backend we can kill' as a core bet but stop at record-level sync. A specified, transport-agnostic encrypted container makes longevity a structural property a buyer can verify, neutralizing abandonment fear in a way Paste (iCloud-locked) and vendor-cloud tools cannot._
118. **Capability-honest UI: per-platform 'what works here' badges as a first-class surface** `[S]` - A persistent, glanceable panel that states exactly what vbuff can and cannot do on the current OS/compositor (e.g. 'GNOME Wayland: capture-on-summon only, no per-app exclusion, no auto-paste') instead of silently degrading. _Value: Silent capability gaps are the #1 source of Linux/Wayland frustration and false-safety (per-app exclusion that quietly does nothing). Every competitor either fakes capability or fails silently; being the tool that is loudly honest about limits is both a trust moat and a direct answer to the architecture doc's Wayland risks._
119. **Two-way live migration importers that adopt a rival's history and keep it updated** `[M]` - Beyond one-shot import, continuously ingest supported Ditto/CopyQ/Maccy/Win+V records with an import receipt for every accepted, skipped, and unsupported item. _Value: Switching cost and abandonment fear are real adoption barriers; a dry-run, reversible importer lowers trial risk without promising recovery of records a rival never stored or exposes incompletely._
120. **Deterministic, user-inspectable sync conflict resolution with a merge view** `[M]` - When the same logical clip is edited/pinned on two devices, surface a visible, explainable resolution (Lamport-clock + LWW shown to the user) with a one-click 'keep both' rather than the silent .conflict-file corruption that sank Ditto-on-Drive. _Value: Fragile, opaque sync is the universal complaint against Paste (iCloud), Win+V, ClipboardFusion, and 1Clipboard. Making conflict handling transparent and never-lossy turns the hardest engineering problem into a trust differentiator, and it is the natural payoff of the sync layer the docs already plan._
121. **Cross-device clip provenance and chain-of-custody** `[M]` - Stamp each clip with verifiable provenance (origin device, source app, copy time, which devices it has synced to) and let users filter and audit 'where has this clip been', with sensitive items flagged if they ever left a device. _Value: For privacy-conscious and team users, knowing a secret-bearing clip's travel history is genuinely novel; no clipboard manager tracks chain-of-custody. It compounds with E2E sync and the privacy ledger into a coherent 'private-by-default and accountable' story that vendor-cloud tools cannot tell._
122. **Reproducible-build + signed-history trust kit** `[L]` - Publish reproducible builds and let the app cryptographically attest that the running binary matches the audited open-source release, with a one-click verification, positioning vbuff as the clipboard manager you can prove isn't exfiltrating your clips. _Value: A clipboard manager sits on a user's most sensitive data; 'open source' alone (Ditto, CopyQ) doesn't prove the shipped binary matches the source. Reproducible builds + attestation is a security moat no competitor in this space attempts, and it directly answers the 'phones home / telemetry from a tool over your secrets' complaints against ClipboardFusion and CopyClip._
123. **Self-destructing and 'view-then-burn' clips for secrets you must move once** `[M]` - A clip class that is captured, synced E2E to one target device, pasted exactly once, then provably wiped (secure-delete + WAL scrub) everywhere, for moving a one-time token or recovery code between your own machines. _Value: Today people paste secrets through Slack-to-self or plaintext history and they linger forever. A first-class ephemeral-secret transport that guarantees single-use and cryptographic erasure is a concrete 'only vbuff does X' for the exact moment users most distrust clipboard managers, and it builds on the secure-delete infra the docs already require._
124. **Per-collection key separation (true vault tabs), not just whole-DB encryption** `[M]` - Let a collection be sealed under its own key/passphrase so even an unlocked vbuff cannot reveal it until that vault is opened, distinct from the full-DB SQLCipher key, with vault contents excluded from sync unless explicitly allowed. _Value: Whole-DB encryption protects a stolen disk but not a shoulder-surfer at an unlocked session; a separately-keyed vault tab raises the bar to 'even running vbuff can't show this without a second secret'. It out-privacies every rival (CopyQ's GPG tabs are clunky and plaintext-by-default elsewhere) and serves the credentials/2FA use case directly._
125. **Capture-and-latency SLO surface** `[M]` - Expose observed captures, intentional skips, detected sequence gaps, p99 search latency at 100k items, idle CPU, and time-to-hotkey-after-login, backed by release gates. _Value: "never silently loses an observed state and never hides an API gap" is testable; "captures every physical copy on every OS" is not. The same surface makes search and startup regressions visible._
126. **On-device-only AI with a hard 'no-network' attestation badge** `[L]` - Offer summarize/translate/semantic-search strictly on-device (and prove it) with a visible badge that the AI path made zero network calls, so privacy users can use AI on clips without the trust tradeoff cloud-routed rivals impose. _Value: Every AI-clipboard entrant (Recopy, PowerToys AI, Paste-via-Apple-Intelligence) raises 'are my clips being sent to a model provider'. Making on-device-and-provably-offline the default and the badge turns AI from a privacy liability into a differentiator consistent with vbuff's private-by-default posture, unlike cloud-routing competitors._
127. **Anti-exfiltration tripwire: detect and warn when another process scrapes the clipboard** `[L]` - Monitor for other processes repeatedly reading the system clipboard (clipboard-watching malware / second managers) and surface a clear alert, optionally auto-engaging incognito, turning vbuff into a clipboard-security sentinel. _Value: Clipboard-scraping malware and dueling managers (Maccy vs Pure Paste double-capture) are real, named threats with zero tooling today. A clipboard manager that also defends the clipboard is a category-expanding 'only vbuff does X' that reframes vbuff from convenience tool to security tool, deepening the privacy moat._

### Monetization & business model

128. **Source-available time-delayed open license (BSL with auto-expiry to MIT)** `[S]` - License the sync/relay server and Pro client code under a Business Source License that automatically converts each release to a permissive license (MIT/Apache) on its 4th birthday, so the code is always source-available and guaranteed to outlive the company. _Value: Directly answers the graveyard fear (ClipX, Clipt, 1Clipboard): users can verify every Pro line is open or will become open, neutralizing the 'dead backend you can kill' objection while still protecting near-term revenue from a competitor reselling your relay._
129. **Posthumous-release dead-man's-switch escrow clause** `[S]` - Publish a signed, legally-binding commitment that if the project is abandoned (no release in N months / company dissolution), the full proprietary sync/Pro source and signing keys are auto-released to the public under a permissive license via a third-party escrow. _Value: Turns abandonment risk into a marketing asset: in a category littered with dead tools, a contractual guarantee that vbuff cannot strand its paying users is a differentiator Paste, Ditto, and TextExpander cannot match._
130. **Pay-once-per-major-version with grace-window updates (no recurring sub for local features)** `[M]` - Charge a one-time price for a Pro major version (e.g. v1.x) that includes 18 months of updates, then charge a fair upgrade fee only for the next major version, with the previous version remaining fully functional forever. _Value: Sidesteps the perpetual-to-subscription betrayal that burned Paste/TextExpander users while still funding ongoing dev; the 'old version keeps working' promise is uniquely credible because local features need no server._
131. **Split billing: local Pro is one-time, only the relay is subscription** `[M]` - Make every device-local feature (FTS5/fuzzy/regex search, transform palette, snippet expansion, paste stack, rich previews) a one-time purchase, and charge recurring fees ONLY for the hosted internet relay that has genuine ongoing server cost. _Value: Aligns price with actual marginal cost and customer mental model: people resent subscriptions for software that runs on their own machine but accept paying for a service that costs you bandwidth every month, defusing the #1 pricing grievance in the category._
132. **Bring-your-own-relay credit that fully zeroes the sync subscription** `[M]` - Anyone who self-hosts the open relay gets internet sync at no recurring charge forever; the paid hosted relay is positioned purely as 'convenience hosting' for people who don't want to run a server. _Value: Keeps the privacy/self-host promise honest (paying is never required to get any feature, only to avoid ops work) and converts the self-host tier from a revenue leak into a trust signal and a funnel of advocates._
133. **Zero-knowledge metered relay billing (bytes relayed, not clip contents)** `[L]` - Bill hosted-relay usage on ciphertext bytes/devices relayed - quantities the server can legitimately measure - and publish that the billing pipeline provably never sees plaintext, item counts, or clip types. _Value: Lets vbuff charge for the cloud service without contradicting 'never leaked': the billing model itself becomes proof of the zero-knowledge claim, versus competitors whose usage-based plans imply the vendor inspects your data._
134. **Founding-license price lock with public, dated price ledger** `[S]` - Sell a limited 'founding' Pro license that locks the buyer's price (and renewal price, if any) permanently, backed by a public append-only changelog of every price the product has ever charged. _Value: Pre-commits against future price-gouging in writing, which is exactly the trust gap left by tools that quietly hiked prices; early adopters fund the risky pre-sync phase in exchange for a guarantee they'll never be repriced._
135. **Team tier billed per shared-snippet-library, not per seat** `[M]` - Price team/collaboration on the number of shared snippet libraries or shared pinboards an org maintains, with unlimited members per library, instead of TextExpander-style per-user seats. _Value: Removes the per-seat tax that makes TextExpander expensive and discourages adoption; small teams and open-source communities can share a library for a flat fee, and the metric (libraries) is something the zero-knowledge server can count without reading content._
136. **Paid signed/notarized binaries, free build-it-yourself path** `[M]` - Keep the full source buildable for free, but charge a modest fee for the code-signed, notarized, auto-updating prebuilt binaries (Apple notarization, Windows EV cert, signed updates). _Value: Monetizes real recurring costs (Apple Developer fee, EV certificate, build infra) that OSS clipboard tools quietly eat, while preserving the 'free if you compile it' OSS contract - a model that funds the trust-critical signing chain attackers most want to compromise._
137. **Verifiable-build bug bounty funded by a slice of relay revenue** `[M]` - Earmark a fixed percentage of hosted-relay revenue for a public security bounty specifically targeting the at-rest encryption, the zero-knowledge relay, and reproducible-build verification, with results published. _Value: Converts revenue into auditable trust on the exact claims vbuff sells (encrypted-at-rest, never-leaked, no backend that can read you); a clipboard tool sits on users' most sensitive data, so funded external scrutiny is a defensible reason to pay._
138. **Sponsor-funded feature bounties with public ledger (no closed-source gate)** `[M]` - Let users and companies crowdfund specific roadmap items (e.g. a new platform port, a transform) via a public bounty board; funded features ship into the open core for everyone rather than behind a paywall. _Value: Provides a revenue stream that grows the free product instead of fragmenting it, matching the OSS-trust positioning; companies that need a Linux/Wayland fix can pay to accelerate it without vbuff having to invent an artificial Pro gate._
139. **Privacy-preserving paid 'compliance config' for regulated orgs** `[L]` - Sell a managed enterprise policy bundle (forced encryption, mandatory exclusions, retention floors, disabled export, audit-of-config-not-content) as a paid add-on that, by design, gives admins control without ever exposing employee clip contents to the vendor or the admin. _Value: Targets the security teams that BANNED Ditto for plaintext storage - a concrete, paying buyer - while keeping the zero-knowledge promise to end users intact, monetizing governance rather than surveillance._
140. **Generous free tier capped on devices-synced, never on local history** `[M]` - Make unlimited local history, search, and all local power features permanently free; gate only the number of devices in an internet-synced mesh (e.g. 2 free, more = paid), since cross-device reach is the value sync uniquely adds. _Value: Avoids the resented 'your own history is paywalled' trap and the OS-floor erosion problem, charging only for the multi-machine reach that is genuinely hard to build (NAT traversal, zero-knowledge relay) and that no free competitor delivers over the internet._

### AI / ML product opportunities

155. **Local embedding model as a hot-swappable platform trait** `[M]` - Define an EmbeddingBackend trait (alongside the four existing platform traits) so semantic features bind to a model interface, not a model, letting users pick bundled ONNX/GGUF small models, a no-op stub, or an external endpoint per device capability. _Value: Semantic search (#95) is already planned but undecided on the model; making it a trait keeps vbuff-core pure and testable with a mock embedder, lets low-RAM Linux boxes opt out cleanly, and avoids shipping one fat model that bloats the binary. Architectural enabler for every other AI idea here._
156. **Embeddings indexed at capture, stored in the encrypted CAS not a separate index** `[M]` - After SQLCipher and encrypted CAS storage are wired, compute the clip embedding inside the same txn-per-capture pipeline and persist the float vector as an encrypted CAS blob keyed by content_hash so it never lands in a plaintext side file. _Value: Most semantic-search add-ons leak a plaintext vector DB next to an encrypted store. Reusing content_hash would deduplicate embeddings, and a future canary-grep release test must explicitly cover the DB, WAL, and CAS; the current unencrypted store cannot satisfy that test._
157. **Privacy-tiered AI: per-clip 'AI eligibility' flag derived from the capture-decision gate** `[S]` - Extend the existing fail-closed capture gate to also emit an ai_allowed bit (concealed/secret-detected/excluded-app clips get ai_allowed=false), and have every embed/summarize/OCR path hard-refuse clips where the bit is false. _Value: Turns the recommendation's 'opt-in, off by default' AI stance into an enforceable invariant instead of a setting people forget. A copied password or JWT is structurally barred from any model, on-device or not, which is exactly the Clipory failure vbuff markets against._
158. **On-device intent classifier that routes clips to actions, not just tags** `[M]` - A tiny local classifier maps each clip to a likely next-action (open-as-URL, run-as-shell, paste-as-plain, decode-JWT, convert-color) and surfaces the top action as a one-keystroke default in the popup. _Value: Differentiates from rule-based auto-tag (#192/238) and from passive type detection (#170): it predicts what the user will DO, collapsing the transform palette into a single contextual suggestion. Fits the 'approachable, one-click actions' wedge (Bet 4) without a scripting DSL._
159. **Smart paste that adapts the clip to the destination app's expected format** `[M]` - Detect the focused target at paste time (already captured for paste-back) and apply a learned format policy, e.g. strip rich formatting into terminals, convert HTML to Markdown into editors, flatten to plain into chat, using a small on-device model over the existing per-app paste-rules. _Value: Per-app paste rules (#130/481) are manual; this learns them from the user's accept/undo signals so 'paste' just does the right thing per app. Highest-frequency daily action, and the focus token is already in hand from the paste-back design._
160. **Clip threading: group related clips into sessions via embedding + temporal clustering** `[L]` - Cluster consecutive clips that are semantically and temporally close into an inferred 'work session' (e.g. all clips while assembling one email), shown as a collapsible group in history. _Value: Solves the real pain of history-as-undifferentiated-firehose without manual tagging or folders. Reuses the embeddings already computed, plus the timestamp already stored. Novel: no catalogued feature groups by inferred task, only by type/tag/app._
161. **Natural-language history query over the local socket and CLI** `[M]` - Add `vbuff ask "the staging db connection string I copied last week"` that runs an on-device retrieval (FTS5 + embeddings) and returns ranked clips as JSON, exposed through the existing IPC/CLI client. _Value: Extends planned semantic search (#95) into the headless/automation surface the architecture already has (vbuff-cli, control socket), where power users and scripts live. Concrete vbuff fit: it is a pure IPC client verb, no GUI work, and dogfoods the same retrieval the popup uses._
162. **On-device PII/secret detector that augments redaction with model confidence** `[M]` - Combine the planned regex secret detectors (#336) with a small NER-style local model so it also catches free-form sensitive text (a pasted home address, a person's medical note) that regex misses, feeding the AI-eligibility flag and shorter retention. _Value: Regex catches structured secrets (cards, JWTs) but not unstructured PII, which is the larger real-world leak surface. Strengthens Bet 3 (private by default) and directly hardens what gets excluded from sync and from any AI path._
163. **Do not sync embeddings before both ranking and replication earn promotion** `[L]` - Treat encrypted cross-device embeddings as a separately gated future experiment: contextual ranking must first beat the deterministic baseline, M9 handoff must pass, and ambient replication must later receive its own threat model. _Value: shipping vectors through a contract-only sync crate would compound two unproven systems, leak new metadata, and spend effort on mobile-scale recomputation before either users or measurements justify it._
164. **Deduplicate by meaning, not just by bytes (near-duplicate collapse)** `[M]` - Beyond BLAKE3 exact-hash dedup, optionally collapse clips whose embeddings are within a cosine threshold (the same URL with a tracking param, the same paragraph re-copied after a one-word edit) into a single entry with variants. _Value: Exact-hash dedup (#40) misses the most common real clutter: trivially-different re-copies. Cuts history noise that drives the 'firehose' complaint, and reuses embeddings already on hand. Opt-in to avoid hiding intentional variants._
165. **Explain-this-clip: on-device summarize/decode for opaque content** `[M]` - A context action that, for a selected clip, produces a one-line local explanation: decode+describe a JWT's claims, summarize a long pasted log into its error, or describe what a base64/hex blob decodes to, rendered in the preview pane. _Value: Sharpens the catalogued summarize/rewrite (#197) and JWT decode (#185) into a single 'what is this?' affordance aimed at developers (the delight segment). Keeps data local, fits the per-type preview panel (#158) already in the UI plan._
166. **Model and inference budget controls with battery/thermal awareness** `[M]` - A settings panel governing AI: which model, max RAM, run-on-AC-only, defer-embedding-while-on-battery (mirroring the existing battery poll throttle #650), and an inference queue that drains during idle. _Value: On-device models are the privacy story but also the perf/battery risk; making the cost visible and bounded prevents the Maccy-style 'clipboard app pins a core' reputation. No competitor exposes an AI compute budget; it reinforces the trust/longevity wedge (Bet 5)._
167. **Active-learning auto-tagger trained from the user's own tagging actions** `[L]` - When a user manually tags or pins clips, treat those as labels to fit a lightweight on-device classifier that proposes tags for future clips, with accept/reject feedback refining it, all data staying local. _Value: Auto-tag rules (#192/238) require users to author rules; this learns the user's personal taxonomy with zero rule-writing, which is the approachability bet. Personalized and private by construction since the training set never leaves the device._
168. **Image clip auto-captioning for accessible, searchable thumbnails** `[L]` - Run an on-device vision-caption model over image clips (opt-in, gated by ai_allowed) to produce a short alt-text caption stored as metadata, feeding both screen-reader output (#604) and search, distinct from full OCR. _Value: OCR-everything is the catalogued privacy footgun; a short caption ('a dark UI screenshot') is far lower-risk than transcribing every secret in a receipt, yet still makes image history searchable and accessible. Complements rather than restates #94/163/164/604._
169. **Hijack-aware paste guard using semantic mismatch detection** `[M]` - Extend planned clipboard-hijacking detection (#363) with a model check at paste time: if the about-to-be-pasted content's type/semantics differ sharply from what was copied (a copied crypto address now a different address), warn before injecting. _Value: Pure byte-diff hijack detection (#363) misses sophisticated swaps; a semantic guard catches address-swap and lookalike-domain attacks at the moment of harm. A security-flavored AI use that reinforces Bet 3 and is genuinely novel among clipboard managers._
